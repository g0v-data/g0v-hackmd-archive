# [OCF] 開放資料平台增能，強化回饋蒐集能力，進而提升運用便利性
## 問題界定
政府資料開放平台雖具備資料匯集、活化運用展示及建議回饋功能，但卻未檢測資料集內容是否符合於所訂定的各式行政命令或指引，並且於資訊頁面標示合規狀態。現存狀態無法讓使用者確認資料揭露之內容是否合規，恐導致開發者因擔心觸法而降低使用意願；另一方面，活化運用專區的各項示範雖然標示其使用之資料集，但卻未揭露其使用該資料集之感受好壞，此況不利於開放資料運用之推廣。
## 承諾事項內容
首先，政府資料平台各資料集的說明頁中，應增加欄位註記資料集是否符合相關命令、規範、指引及評分標準。
其次，單筆資料集頁面應增列「使用回饋」欄位，並提供回饋收集管道。此欄應明確列出好的資料集之優點，不好的資料集可以怎麼改進。（此部份可參考韓國 Code for Korea 「全國公共資訊地圖」專案網頁）。具體執行作法，可以先要求後續黑客松競賽或資料應用競賽團隊，提供單筆資料集使用上的「正反面」意見。同時鼓勵更多活化運用政府資料的使用者回饋，
以上新增欄位，最終成果：單筆資料集說明頁中，除了基本資料外，亦可呈現該資料集合規狀態及使用心得等資訊。令民眾可以安心運用，同時亦提供資料集維護單位作為後續品質提升的參考依據。
同時，數位發展部應先依所公開相關命令、規範、指引及符合新評分標準，提出其實際運用資料案例作為示範案例，示範內容須含最佳案例與揭露可改進方向等項次。
最後，基於上述機制調整與成果，辦理培訓課程與說明會，教育政府機關第一線執行者及中階長官，以利數位發展部的整體策略及其示範案例得以傳達到各部會。
