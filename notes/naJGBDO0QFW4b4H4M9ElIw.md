## g0v summit

> Cofacts 是 2016 年在 g0v 黑客松提案的開源專案，是能自動幫你查証不實訊息的聊天機器人，目前累積超過 37,000 則回應的可疑訊息資料庫，發送過成千上萬則查核回應給對訊息感到困惑的朋友。
>
> 程式碼與所有文件資料通通開源，想要照做一個事實查核的機器人都可以閱讀 Cofacts 的教學文件，在你的筆電上自己做一個唷！接資料庫也沒問題，例如美玉姨、例如趨勢科技防詐達人，Cofacts 讓你 chatbot 自己做，還可以幫助朋友事實查核， Cofacts 就是想當你在 LINE 上最好的朋友。
>
> 人人都看過假新聞，但錯誤的訊息到底是怎麼影響我們的生活、經濟、民主、社會呢？Cofacts 會在一個半小時的工作坊帶參加者分組解析 2020 大家會收到的焦點訊息。例如總統選舉、新型冠狀病毒、高雄市長罷免 時期所遇到的不實訊息，研究這些特定時期不斷出現的大外宣小內宣， 找出不實訊息散布的樣貌與反制的方法 。
>
> (不需要具備任何程式基礎)
> 
> [https://summit.g0v.tw/2020/agenda/2020-12-05/5ee48620ba206b030d66a2e5]

- 編輯公約 + complaint filing + 折疊 [name=mrorz]
    > 提升 accountability 計畫：
    > 1. 發布 Cofacts 編輯公約，明確記載 IFCN code of principles，亦即：(1) checking all sides, (2) revealing how conclusions are made, (3) agree to be contacted by Cofacts when someone files a complaint, (4) agree that their work is published under open license
    > 2. Cofacts 編輯若想簽署此公約，必須提供可以我們聯絡得到他的方式。
    > 3. 若編輯同意簽署編輯公約且能聯絡得到，就在 editor profile page 與他們撰寫的回應上加註「已簽署 Cofacts 編輯公約」
    > 4. 若編輯沒有簽署公約，他們的回應就會比較難被看到（可能摺疊之類的）
    > 5. 建立 complaint filing mechanism，讓讀者可以針對回應 file complaint
    > 5-1. file complaint 的人必須要指出回應到底不符合公約的哪個部分。
    > 5-2. 若回應作者有簽公約，我們就會將 complaint 內容 pass 給編輯，讓編輯限時改進；另外，在給編輯之前，我們會拿掉 complaint 裡情緒性與攻擊性字眼，保護 Cofacts 編輯的心理健康。
    > 5-3. 若回應作者沒有簽公約或有簽公約但沒有在時間妹為應，就會標注該回應已遭 dispute，且讓回應比較難看到（被折疊或刪除）。
    >
    > Prerequisite:  Cofacts 網站要先做好 profile page、若 badge system 能先完工就更好了。
