任一份文件（document）的協作翻譯機制，預想會有兩個回合

### 第一回合：華、英 2 版本先確立

- submit 華語或英語文件
- 產生 unique key
- 加入列表
- 開放 submit 翻譯完成的華語或英語文件，允許多個版本
- 經過某編審機制後完成

可能需要
- 原作者註明原稿是否可以修改
- 制定一群文件的 context，避免翻譯偏移脈絡

## 建立專案 README 模板

目標：撰寫類似 readme 的介紹模板，讓想貢獻的其他語言使用者能夠快速進入狀況並貢獻。

機制：本內容可以放在共筆前，由專案參與者填寫。完成填寫後就可以填寫表單 （類似 covid-19 的translation 機制）

表單連結：https://docs.google.com/spreadsheets/d/12cwWiZUXrSXR2HMvwuy6bY7ykn3WMufFWC_WDFK9IYE/edit?usp=sharing 

對象：初步以漸趨成熟的專案為主，邀請更多專案坑主加入協作，剛開始協作的專案也可以此為方向前進。

內容模板：（需要大家看看）
1. 本專案的目標（一句話）ex. 讓更多人參與法規修訂的討論。
2. 目前正在解決的問題：ex. 處理實聯制的法律問題共筆，需要有人幫忙擴散討論。
3. 需要的技能：ex.  社群經營
4. 是否有定期的討論或聚會時間：有 / 無
5. 若有，頻率與下一次時間？ ex. 一週一次 / 下週一晚上7:00-9:00
6. 若無，目前規劃的討論時間？ex. 5/31 7:00-9:00
7. 專案頻道： #vtaiwan (若無頻道？[name=peter]）
8. 聯繫人及方式：不一定要是坑主 ex. peter
