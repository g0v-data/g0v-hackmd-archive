# g0v x VUSE 頻道合作

> [點此觀看原始內容](https://g0v.hackpad.tw/jAz31CKxsch)


## 概要

g0v 在和 NGO 團體合作的過程中，發現大多數小型 NGO 亟需設計資源將訴求與論述圖像化、幫助理念的傳播，但除了熱情之外沒有經費。另外，設計師來到 g0v 也是滿腔熱血，但卻不知道要如何幫忙。g0v 和 VUSE 聯絡，初期希望可以開放專用頻道給 g0v 與 VUSE 的設計師認領訴求與 UI 設計需求、產出圖畫，並以開放的授權方式供多次使用。


## g0v 需求

Tuiry 指出，社運圈很難精確描述出想要的感受，設計師需要花時間查資料與溝通，才能理解論述，並且使產出的作品具有能被社運圈所用的功能性（表達訴求）與擴散性。

案主與設計師之間的溝通，若有一個專業的 PM 熟悉雙方的語言與做事方式，在溝通之前讓兩方知道對方要的是什麼，溝通會好很多；否則，雙方的熱情將會消磨在往復的修改之中。

溝通成本很大的時候，同時也會造成「分手」的困難。願意無償幫忙的熱血設計師，以及想要推廣訴求卻缺乏經費的 NGO 憑藉著一股熱情開始合作，但若在許久的溝通之後發現雙方不太合拍，設計師往往會因為人情與對方的宣傳壓力而難以將案件脫手，只能挫折地繼續做下去；反過來講，如果溝通之後可以能夠了解彼此，那就很有成就感、得到非常愉快的合作體驗。

venev 則認為，目前參與 g0v 的設計師，由於 g0v 沒有明確的「參與」步驟，不知道要怎麼做。NGO 的需求橫跨視覺設計（海報傳單）、插畫，而 g0v 本身也有不少專案需要 UI design。NGO 與 g0v 成員們大多無償參與，對於無償幫忙的設計師，要求並不高；但設計師與團體成員溝通與將文字圖像化的過程，也可以用在業界打交道，對設計師來說可以累積作品與經驗；Tuiry 也認為這樣的機會很適合新手設計師磨練。


## VUSE 現符合需求之處

- VUSE 服務核心就是文字免費轉為圖畫，理念重合。
- VUSE 平台提供新手設計師舞台、觀眾與作畫靈感，讓設計師練習用圖畫說故事。
- VUSE 希望平台文字、作品多元化，期盼能用圖畫來紮實記錄生活與社會的不同面向。
- 頻道可收集並陳列圖畫與文字訴求。
- 設計師可以收藏頻道獲取通知、可以認領文字訴求。
- 圖畫頁面下方有討論區可供討論。
- 平台內容有 REST API 可供存取，故 g0v 設計師創作之內容不會就此鎖在 VUSE 平台。(註：流量考量，不開放 CORS，請用 backend crawler 爬取)
- 和綠盟合作過，曾經有依訴求文字產出圖畫的經驗，也曾處理過 Licensing 問題。
> Ex: [他們其實很限量!!! ](http://goo.gl/m2wC3P)by [保麗](http://vuse.tw/users/%E4%BF%9D%E9%BA%97)  是與綠盟合作的「[100 個廢核的理由](http://vuse.tw/channels/%E5%BB%A2%E6%A0%B8%E7%9A%84-100-%E5%80%8B%E7%90%86%E7%94%B1/arts)」頻道的作品之一，圖像採 CC-BY-NC-ND 釋出。
> [name=Johnson L]



## VUSE 可能不符合需求之處

- 沒有解決到「分手的困難」。或許在認領前，一個匿名的討論串讓設計師問問題、確認雙方是否情投意合，是一種「分手困難」的解。
- Ian 表示，NGO 圖的需求是「一望即知」的議題表達能力，在圖片 go viral 的同時也能表達訴求。目前平台設計師則習慣於軟性的心情文字。
- g0v 需求為 UI design，NGO 的需求除插畫外大多屬於平面設計，和 VUSE 作品不合，UI design 與設計作品可能會不敢發。
- 依照 VUSE 平台的心情文字，若 VUSE 提供社會議題頻道，最後會不會變成大腸花呀 (´･ω･`)
> 我的確有在g0v畫過一些黑特作品
> [name=Tuiry H]

- 圖片會壓縮，原始檔需要另外提供連結。
- 或許很多設計師都有興趣提供草圖來討論，但最後只有一名設計師可以認領。
- 看圖介面並非對設計或 UI design 作品最佳化，長的文字會很長。


## 擬合作模式


### VUSE 方面

1.  開 g0v 頻道，整理文字訴求。
> [http://vuse.tw/channels/g0v](http://vuse.tw/channels/g0v)
> [name=Johnson L]

2.  將 g0v core collaborators 加入頻道管理員。
> [http://vuse.tw/channels/g0v](http://vuse.tw/channels/g0v)
> [name=Johnson L]

3.  將 g0v core collaborators 加入 VUSE admin 權限（比照綠盟合作之管理員），能進入 meiling 後台修改想法與設計師資格，但無法做其他變更。
4.  補上 API documentation。
> [http://vuse.tw/dev](http://vuse.tw/dev) , 測試網站： [http://vuse-dev.herokuapp.com](http://vuse-dev.herokuapp.com)
> [name=Johnson L]

5.  在投遞 g0v 想法蛋時，加註 Licensing 資訊、「設計師想知道什麼？」和「NGO 想知道什麼？」等等合作 SOP。
6.  在認領之前鼓勵設計師多多討論。
7.  鼓勵 NGO 回饋「圖畫在表意方面，哪裡有做好哪裡沒有做好」並公開，供設計師們觀摩學習。
8.  補完頻道權限（僅讓管理者投稿，未來讓每個 NGO 都登記為頻道管理者）

### g0v 方面

1.  將 g0v 設計師導到 VUSE，由 g0v core collaborators 設成設計師資格，或在寄信時加註「來自 g0v」可以快速通過。
2.  決定 g0v 頻道名稱，設定 g0v 頻道大頭照、封面照片、介紹文字。


## 合作觀察重點

- 產出圖片與訴求的契合度
- NGO 回饋的質量、設計師的反應
- 剛性訴求文字是否會和柔性心情文字互相排擠？（數量上的增減）
- 有多少插畫、多少視覺設計、多少 UI design 成品？
- 孵畫時間是否符合議題進程？
- 討論是否充分？
- 設計師比較習慣和 g0v 溝通，還是和社運團體溝通？

以上觀察重點可以用在改進 VUSE 平台本身，或者是開新 open source project 的重要參考。
