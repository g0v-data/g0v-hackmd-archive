# DigiResiThon 數位韌性松 bootstrapping

## 目標
- [ ] 釐清在台灣對外網路中斷的情況下，有哪些線上服務或資訊是需要正常運作的？
    - [ ] 準備各地區可以鏡像的關鍵資訊網站，以防島內網路受損。
    - [ ] 政府的緊急通知網站
    - [ ] 購買民生必需品的線上平台
    - [ ] 通訊工具
        - [ ] PTT 
        - [ ] [g0v.social mastodon](https://g0v.social)
        - [ ] [keet.io](https://keet.io/) / [Quite](https://tryquiet.org/)
- [ ] 評估線上平台和服務是否在台灣內部有足夠的冗餘，釐清其是否能獨立於海外網路，以在台灣對外連線中斷的情況下繼續運作
    - [ ] 對企業的應變計劃進行調查
        - [ ] 像 Google 和 Cloudflare 這樣的服務在當地有基礎設施，
        - [ ] 由網路購物平台，如 PChome 和 7-11 等公司提供的像物流和供應鏈這樣的關鍵民生需求，在斷線期間仍然是必需的。
    - [ ] 甚至是將某些線上服務作為關鍵基礎設施，應要求其加入年度漢光演習，以在模擬緊急情境中測試他們的系統連線能力
    - [ ] NGO 可以調查線上在斷線期間維持服務的能力並發布報告。
        - [ ] 要把報告標準寫出來
        - [ ] 列舉企業名單
- [ ] 引入其他國家或組織在斷線期間維持數位服務的經驗或解決方案，確保台灣島內的 connectivity
    - [ ] 技術解決方案
        - [ ] 像是 Mesh Network 這類的替代通訊方案
            - [ ] 案例蒐集，開一個頁面整理頻道中已張貼的 美國、德國案例
        - [ ] 和很多民間解決方案
    - [ ] 經驗分享
        - [ ] 馬祖經驗，台灣實現(X) [馬祖斷網 60 天歷史紀錄共筆](https://g0v.hackmd.io/A-LPXr1RTiSK_iUwJsJowg)
- [ ] 制定計劃，讓民眾在斷網前後都有辦法知道有哪些可用的緊急線上服務或方案
    - [ ] 對於廠商加入的動機：向本地使用者 sellout 斷網的解決方案
