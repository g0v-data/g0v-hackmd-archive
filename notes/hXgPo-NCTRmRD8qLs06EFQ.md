# 公民行動方案：南港小基站錯誤時區案例
g0v slack @Teemo 

---

## 🎯 問題背景

* 南港某處 4G/5G 微型基地台設定錯誤
* 導致手機經過時自動跳到 UTC+7 時區
* 此狀況可能影響手機時間同步
* 造成影響
    * 金融交易、通訊定位、醫療等服務異常
* 需要行動 
    * 公民檢測 → 蒐證 → 通報 → 追蹤 的方式
    * 促使管理單位修正

---

## 📝 行動步驟

* 現場檢測：手機 App / SDR 測量 → 確認問題。
* 精準定位：必要時用 SDR + 定向天線比對。
* 證據收集：表格填寫 + 照片 + 截圖。
* 對外通報：電信公司客服 + NCC 1915。
* 後續追蹤：定期複測，更新結果，若未改善 → 媒體/民代關注。

---

## 🔄 流程圖 (Mermaid)

```
flowchart TD
    A[現場檢測<br>手機 App / SDR] --> B{是否發現時區異常?}
    B -- 否 --> Z[結束<br>無異常]
    B -- 是 --> C[精準定位<br>記錄 Cell ID / PCI]
    C --> D[證據收集<br>表格 + 照片 + 截圖]
    D --> E[對外通報<br>電信公司客服 + NCC 1915]
    E --> F{業者是否修正?}
    F -- 是 --> G[後續追蹤<br>再次檢測確認]
    F -- 否 --> H[擴大行動<br>媒體/民代關注]
    G --> I[完成案例<br>建立公民觀測紀錄]
    H --> I

```

## 檢測紀錄表格欄位

[檢測紀錄表格欄位](https://docs.google.com/spreadsheets/d/1UnsvipspycLtxHvb54R4JolBNVOau0sX/edit?usp=sharing&ouid=104830429551806592903&rtpof=true&sd=true)

|欄位	| 說明 / 範例|
|--------------------------|--------------------------------------|
|檢測時間	|請使用當地時間，格式 YYYY-MM-DD HH:MM:SS|
|裝置類型	|手機 / SDR(軟體無線電) / 頻譜分析儀等|
|Cell ID / eNB ID / gNB ID	|手機或 SDR 工具可抓到的識別碼|
|RSRP / RSRQ / SINR	|LTE/NR 量測值，以 dBm 或 dB 為單位|
|觀測到的時區	|手機顯示的時區（若有跳到 UTC+7 請明確記錄）|
|重複出現?	|是否可在不同時間或來回經過時重現|
|步驟重現方式	|簡述如何重現（例如：從南港路口往東走 50m，手機自動跳時區）|
|建議處置	|建議通報流程：先通知所屬電信業者 + NCC 投訴 (1915)|
|證據收集	|建議同時拍攝現場照片、錄影 + 以 App 截圖記錄 PCI / TAC / 參數|

## ✅ 預期成果

* 鎖定問題基地台位置並蒐證完整紀錄。
* 向電信公司與 NCC 提出正式通報。
* 修正問題，減少民眾因錯誤時區產生困擾。
* 累積經驗開源「公民電信觀測行動手冊」。
