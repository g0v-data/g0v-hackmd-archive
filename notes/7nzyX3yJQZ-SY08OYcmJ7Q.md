# 戰時網路發想

假設：
- 戰爭情況大部份民眾在遇到空襲時需要躲在最近的防空洞內，等到空襲解除後再逃出防空洞
- 空襲警報到解除之間的時間最長不會超過 24 小時（此時間假設的越長後面對於電源和資源的要求就會越高）
- 在地下室手機可能通訊不良，而且手機基地台也有斷線網路中斷的風險
- 可能會停電
- 人民需要可信任的即時資訊以對戰情安心
- FM/AM 可能被蓋台散播錯假資訊

發想解法：
- 透過 FM 廣播戰情非通過人講話的廣播傳遞，而是透過類似撥接數據機的技術，將網路封包轉換成聲音傳輸，而是通過傳遞簽證後的封包資料，以方便驗證資料是否可信
- 可以直接傳遞 https://gov.tw/rss （假設的網址）之類的封包內容，用 RSS 格式傳遞，客戶端只要驗證憑證是「政府伺服器數位憑證管理中心 - G1」簽發的，就可以相信該資料是可信的，RSS 格式則是可以加上時間資訊，以便知道最新戰情
- 可能需要同時多個不同網站和不同憑證來源來傳遞資料，以免單點被破解就可能可以傳遞假訊息。
- 透過 Raspberry Pi + FM 訊號接收器，天線拉到地下室以外的地方，就可以將廣播訊息接收進來
- Raspberry Pi 本身也可以當作防空洞內區網 AP 使用，大家可以用手機練上該 AP 等待最新戰情資訊，或者在同一個區網內交換資訊（例如確保物資數量）

需要開發：
- 需要建立 Raspberry Pi + FM 訊號接收器的開源模組，讓各防空洞都要有一組，並加上蓄電設備確認可以撐完空襲警報解除
    - Raspi4 現在強烈缺貨中，是否有用舊手機的替代方案？如果讓電力續航力夠強？
    - [Turning the Raspberry Pi Into an FM Transmitter](http://www.icrobotics.co.uk/wiki/index.php/Turning_the_Raspberry_Pi_Into_an_FM_Transmitter)
- 防空洞內飲水、食物、藥品、生理用品等物資分配系統。
    - 資料可以不需要雲端化，存在本地的 Raspi 即可，達到去中心化的效果

平日演練：
- 確保防空洞內能夠接受 FM 廣播的戰情（平日該戰情 RSS 可以一直發送目前安全的測試訊息或是今日有多少共機擾台等）
- 要有驗證該 FM 廣播訊息是正確的能力
- 熟悉防空洞內物資分類系統使用，並且平日就讓大家知道自己可能會去的防空洞有哪些，該防空洞可能會有多少人躲藏的需求，大家要怎麼分配當要避難時攜帶的物資種類
- 可能需要手機輪流開關機的共識機制
