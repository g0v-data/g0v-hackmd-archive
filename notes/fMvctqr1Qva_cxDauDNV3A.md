## 韓國經驗：如何推展公民科技到公民組織（NPO, NGO, 與媒體）/ How to extend Civic Tech to civic organizations (NPO, NGO, and the media) in KOREA?

why we started these civic tech projects
a project started by Green Party in Korea, we tried to download by failed, because "downloads are limited to a maximum of 100"

so we decided to support them, to help them, e.g. in writing code and reading data

So we're starting a project to build data containers

## 從 .org 到 .g0v － 台灣NGO的網路參與與數位挑戰 / From .org to .g0v －Internet participant and digital challenge of NGO in Taiwan


[http://www.slideshare.net/YingHsunKao/org-g0vngo](http://www.slideshare.net/YingHsunKao/org-g0vngo)

我原本是環境資訊協會第一任工程師，但是三月的時候不小心變成理事長。

2000年的社會氛圍和網路環境，開始有人討論如何從dot.com 變成dot.org
2016年：新的契機?
-政黨輪替: not only 政府, include立法院, 更多的社會運動人士進入政府體系
-IoT: 是否能幫助環境監測
-OpenSource: more tool

環境資訊協會在 2000 年與 2016 年的網路發展比較
- 2000 年：
    - Web 1.0
    - email + IM
    - 環境新聞沒人要報, 僅在生活版上出現
- 2016年：
    - Web 1.0 + FB 粉專 + 2.0外掛
    - email + IM
    - 環境新聞躍入主流
    - 無法突破的電子報訂戶: 二萬上下

網路發展的重要突破
- Google search 的興起：使小眾的環境網路內容與新聞搜尋成為可觸及
- 2000 年自行架設自有[網站](http://e-info.org.tw)
- 2010 年開始有自己的記者，更趨向媒體的型態
- 程式的翻修(大改版)週期約五年，期間要花費六個月且需要外部工程師進入

nobody to somebody
- DNS管理權是難以去中心化的門檻
- Domain name到期沒續約，一定會被註冊走，而且DNS的爭端處理是以正式組織為優先，nobody擁有的
- 需要值班oncall工程師

資金來源
- 幫政府做網站
- 小額募款

on-line and off-line(人的網路）

組織永續面對的問題
-組織內的溝通
-組織外的目標調整
-成為"既得利益者"的衝突: 例子：如果政府把資料全部公開, 原組織在傳統脈絡下作為中介者的力量就可能會削弱，而讓組織原先在的位置受到影響
-數位落差：NGO最好擁有一個好的技術長或資訊長, 了解組織內部需求, 不一定要自己寫code
-系統上線的第一天就是legacy system
（就開始有維運的成本和造成改版的包袱）

NGO 扮演的角色
- 匯聚資訊與力量的平台
- Proxy: 能較快速的回應政府議題
- Agent: 公民代理人

擁抱技術的同時，藉由自身對議題的深入了解做更好的貢獻
