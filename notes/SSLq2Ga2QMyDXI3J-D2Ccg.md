# 論開放協作 g0v 專案使用 Google Drive 的設定方法 – 以 0archive 為例

許多開放協作的 g0v 專案還是會碰到有些檔案不能、無法開放的情況，Google Drive 是個常用的工具。怎麼設定、檔案怎麼放比較好呢？

以 0archive 為例，[0archive 的 Google Drive 資料夾](https://drive.google.com/drive/folders/18uDOhXBoxuiA0gyEy4U-i0ccpN-DN0bN)長這樣⋯

`0archive`
├ `0archive 開放協作 Collaboration`
├ `0archive 公告 Announcements`
├ `0archive - team only`
├ `0archive - shared with A + B`
├ ...
...

## 根目錄
開一個 Anyone with the link can view 的根目錄，用專案名稱命名，越短越好。
【例】`0archive`

因為設定成 Anyone with the link can view，所以這個根目錄的連結可以公開分享，確保所有人看到的根目錄都是同一個。

## 子目錄
在根目錄下開各種子目錄，設定不同的權限。

### 開放協作
Anyone with the link can edit，任何人都可以看也可以編輯，是最大權限。
【例】`0archive 開放協作 Collaboration`

適合放一些希望公開協作、群眾外包的文件。例如，投影片、蒐集資料的試算表。

### 公告
Anyone with the link can view，任何人都可以看但不能編輯。
【例】`0archie 公告 Announcements`

適合放一些希望越多人看到越好，但不希望被專案成員以外的人編輯的文件。例如，新聞稿、標準文件。

### 其他分享對象
與特定對象分享的子目錄，用分享對象來命名。
【例】`0archive - team only`

資料夾名稱用專案名稱開頭，這樣才不會在瀏覽器很多 tab 之中迷路。
