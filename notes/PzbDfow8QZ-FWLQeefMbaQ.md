# LINE BOT 市容幫手

## 當 LINE 已成為反映市政問題的工具時，機關該怎麼辦？

臺北市政府成立「HELLO TAIPEI」供民眾反映各種大小問題，可以用網站、手機 APP 或是電話通報，系統會將問題成立案件，讓對應的機關部門回應及解決，此為機關部門與民眾對口的主要管道。

除了 HELLO TAIPEI，還有一個管道時常反映各種問題——「里長 LINE 群組」。

里長接收到里民的問題後會傳訊息至群組內，希望相關的機關人員能看到並幫忙處理。LINE 比起 HELLO TAIPEI 更能即時反映問題，使用更方便，不像 HELLO TAIPEI 需要加入會員、留下個資、步驟繁多，查詢案件進度還需要編號密碼等。

而且最重要的是——每個人本來就有 LINE 啊。
　　
除了里長之外，一些與地方里長或民代熟識的市民，也容易透過 LINE 向機關人員反映問題。

## 里長 LINE 群組的改善空間

### LINE 是最方便的管道

里長最重要的工作，就是作為里民與政府部門的溝通橋樑，當里民有任何問題都能向里長反映，里長會尋找對應的機關窗口，幫里民解決困難。

各行政區的區公所為了方便里長與機關人員聯繫、快速解決問題，所以各區公所成立里長群組，將所有里長、里幹事及主要的機關人員加進去群組內，反映的問題諸如：哪裡的路燈沒亮、公園需清潔或修繕、道路有坑洞、哪條路骯髒需清掃、水溝損壞、樹木需要修剪⋯⋯等等。

現今對於里長來說，反映問題的管道有四類：

1. HELLO TAIPEI（包含市民熱線 1999 ）。
2. 里長 LINE 群組。
3. 找議員陳情辦會勘。
4. 區公所定期主辦區務會議或其他正式會議。

上述管道，HELLO TAIPEI 操作不方便、議員民代是里長的壓箱寶、正式會議繁瑣不夠即時。

相較之下，「里長 LINE 群組」最簡便，所以容易處理跟反映的問題，里長習慣丟到群組向機關人員反映，但是群組成員太多，訊息紛雜，無法有效管理案件，目前缺點如下：

1. 容易錯失訊息。
2. 不方便追蹤案件是否完成。
3. 有些使用者容易用情緒性字眼，煽動其他人一起攻擊特定機關，使得該機關難在群組上回應說明。
4. 里長傳報的問題若複雜，則群組上里長訊息將沒人回應，訊息石沉大海。
5. 政府部門無法瞭解及管控案件數目及內容。
6. 有人會在群組內傳罐頭訊息。

### LINE 群組的潛規則

現行的里長 LINE 群組有使用的潛規則及相應的缺點：

|排序|方式|缺點|
|:-|:-|:-|
|1|里長看到任何問題時，拍照上傳，講地點及遇到的問題。|地點及內容常不明確，導致政府人員需打電話詢問里長，增加麻煩。|
|2|政府部門的所有人員收到訊息後，立刻（不超過1小時）判斷是否為自己的權責。|**如是**，要立刻回報「 X 局／處 X 單位收到，立刻派員」。**如不是**，將沒人回應案件，里長會發火，區公所才跳出來協調，嚴重的話，會有人使用情緒性字眼攻擊機關，煽動其他人一同攻擊，造成機關人員難回應。|
|3|政府人員處理完成後，於群組內回報「報告 O 里長，X 局／處 X 單位已於 O 月 O 日完成處理」，並回覆照片。|群組訊息量大，里長不一定看得到回報；政府部門想掌握LINE的案件量，但訊息不易統整。|

目前在群組內的機關人員有：里幹事、區公所、民政局、工務局、環保局、交通局、警察局（不一定）及消防局（不一定）。這些機關因為直接涉及工務（如：道路、水溝、公園、行道樹、路燈、山坡等）、環境清潔及交通問題等最易被市民反映，且要求即時性，所以通常是這些機關的業務會常需與里長聯絡。

目前最常被點名的機關有：工務局新工處、工務局公園處、工務局衛工處及環保局。

## 用 LINE BOT 改善群組溝通

為的是要解決現在體驗的三大問題：

1. 引導通報者傳送正確資訊給回報者。
2. 案件能在通報者與回報者之間有效傳遞。
3. 案件需要能被管理。

構想加入 LINE BOT 後的里長群組運作方式 ：

* 現有里長 LINE 群組導入機器人：**通報者**鎖定里長、**回報者**鎖定機關人員。
* 通報者照機器人引導輸入問題相關資訊，機器人統整後，傳給機關人員來回報，如：地點及照片等。
* 回報者能夠針對通報者問題做專一回覆，如：需要的改善時間、改善完成的照片等。

如何判斷哪個問題該分給哪個機關回報？可以參照臺北市研考會為各種市民問題，訂定出各機關分工權責，以及現有政府的開放資料（[臺北市政府行道樹資訊網](https://geopkl.gov.taipei/)、[臺北地政雲](https://cloud.land.gov.taipei/LandDevel/PubActLand.aspx)、[臺北市山坡地資訊整合系統](http://www.geomis.gov.taipei/GEOINFO/NormalPage/GeogisPage.aspx)）。若收到的機關判斷非屬其權責，則回報系統，由系統再通知其他機關，有爭議的話則應另訂協調機制。

由於爭議部分將會使得現階段使用意願降低，所以先選擇沒有爭議的問題。現階段有些較單純的問題可供試運作：

1. 路燈：處理的問題通常為路燈沒亮，管理機關僅臺北市政府工務局公園路燈工程管理處，每天有 24 小時值班人員，如涉及電路問題，則由該機關聯絡台電公司。路燈點位目前已開放[臺北市政府行道樹資訊網](https://geopkl.gov.taipei/)查詢。
2. 公園：範圍算明確，反映事項通常為需修繕、清潔或修剪樹木等，雖然有些地方看起來是公園，但非屬臺北市政府工務局公園路燈工程管理處管轄，相關點位已開放在[臺北市政府行道樹資訊網](https://geopkl.gov.taipei/)、[臺北地政雲](https://cloud.land.gov.taipei/LandDevel/PubActLand.aspx)內。現階段有假日值班人員收案，但非 24 小時值班。
3. 環境清潔：道路及人行道的環境清潔都屬於環保局範圍，但由於範圍廣泛，有時可能非屬環保局的權責，現運作方式為環保局接收通報，再 LINE 轉傳或打電話給相關機關。目前環保局清潔隊屬輪班制，但非 24 小時。
4. 交通號誌：標的及權責機關明確，但是反映的案件較少，而且更需要即時解決問題。

## LINE BOT 的潛能

雖然現行「HELLO TAIPEI」是反映市政問題的正式管道，但是一般市民也可能透過地方里長或民代轉發 LINE 訊息反映問題。

臺北市目前共有 456 里，若里長端能推行成功，未來甚至可以推廣到一般市民，讓反映問題可以更有效率。

當案件都能記錄歸檔，對於現在的里長來說，亦是發表自己政績的紀錄，誘使其使用 LINE BOT。對於公部門來說，可以更精確掌握每個里常見的問題。

上述皆為一般性的案件，另外市府 1999 另外負責「緊急案件」的通報處理，由民眾打電話通報地點及緊急狀況，留下聯絡電話後，再由市府 1999 聯繫機關的窗口，如：消防局、工務局、警察局等。目前緊急案件的通報缺點為描述不精確，沒有照片，有時會有誤報情況，浪費機關的資源。如有辦法將系統擴展使用在「緊急案件」通報管道，將大幅提升政府即時正確解決問題的能力。

## LINE BOT 將面臨的挑戰

### 如何不干預既有使用者習慣

現在的里長仰賴 LINE，有部分原因平時就與其他里民、里長、民代，甚至是機關人員用 LINE 保持關係，LINE BOT 的加入是為了讓溝通效率更好，資料更容易搜集及掌握。

### 如何在公務機關推動起來

公務機關習慣打安全牌，不容易接受新工具。除非有長官或里長推動，否則一般機關不會接受新工具的介入。

另外，現有 LINE 群組窗口內的公務人員，多數人是不領額外薪水的情況下，義務性地回覆里長訊息，不分上下班及例假日。雖然這個狀況是市府該認真檢討的問題，對於里長端來說並非困擾，但如何不增加現有人員的負擔，也是要考量的事情。

### 開放 LINE BOT 給市民後，如何與 HELLO TAIPEI、1999 區隔

現在的 HELLO TAIPEI 操作雖較繁複，但那是因為法規規定需留下真實姓名及資料才可立案。

未來 LINE BOT 若開放給市民後，將面臨相同的法規問題，且用途的目的與現存的 HELLO TAIPEI 相同。
