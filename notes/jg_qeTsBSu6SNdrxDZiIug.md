# 關於 Code for Japan Summit 

公民科技組織 Code for Japan 成立於 2013 年，為日本註冊登記的非營利組織，致力於透過 IT 技術的應用與公民的參與，解決日本各地之地區性的公共議題。並以「共同思考，共同創造」為願景，與日本各地許多地方政府合作，提倡公私協力共同解決社會問題，在 Code for Japan 的協助下，於日本 80 個鄉鎮市各設有獨立的 「Code for X（地區名）」組織，形成日本的「Code for 聯盟（ブリゲード）」，與日本各地的地方創生政策有緊密的連結，Code for Japan 提供 IT 人才，並串連各地資源，促進公民、政府與企業的合作，與台灣 g0v 零時政府同為 Code for All 全球公民科技聯盟成員之一。

為提供日本各地從事公民科技的人們互相交流的平台，Code for Japan 每年固定選定不同城市舉辦 Code for Japan Summit。其特色為主題多、互動性高、地方性強，每年聚集數百位日本各地的公民與公務員參與。 2019 年度的 Code for Japan Summit 以「Spark Joy！」為主題，破除一般人對公民科技高門檻的想像，打造輕鬆的氛圍，讓更多的公民能夠親近公民科技的領域，體現 Code for Japan 「共同思考，共同創造」的精神。 動員日本各地約 90 名的志工及 130 位講者，其中包括剛從日本 Yahoo! 走馬上任東京都副知事的宮坂學、及主辦地千葉市市長熊谷俊人，共計吸引 1,003 人次的參加者，其中半數包含地方公務員，彼此交流分享經驗與推動情形。

# 重點摘要
### Keynote、開場
開場由兩位 Code for Japan 成員擔任主持人，首先說明這次 Summit 所使用的 APP。參與者可以利用 APP 查閱場次內容，並 check in 參與場次，軟體將自動統計並顯示各場次 check in 人數，議程結束後，大會頒獎給獲得最多 check in 次數的場次。此外，為了方便不使用日語的參與者，在各場次資訊頁面中提供 UD TALK 連結，參與者點選連結後，可以直接在手機上看到英文翻譯頁面。介紹人並特別強調，APP 是今年年會的新嘗試，參與者如果有不了解之處，可以向工作人員尋求協助，並請會眾回報好用和不好用的狀況，以便作為日後改進的參考。

Keynote 議程由社會學科出身、參與美國 Code for America 的講者介紹 Open Maine ，如何讓不具備專業技術的一般民眾一起參與公民科技、預算視覺化（類似台灣 g0v 政治獻金 / 投票指南專案），以及與地方政府職員協作、活用 Open Data、共創設計原型到當地做原型測試。 Open Maine 的活動也很多元，有舉辦 Civic Deisgn Fest、由地方 NPO 開始做的專案，也有以不同地方作為會場（包含公園、Art work Space）的活動。參與者有人畫地圖、有人做當地調查等等，大家透過這樣的活動擴大社群連結、交換不同想法，讓一般市民參與協作。企業 WELMO 則是介紹與中央或地方政府合作高齡者長照專案，結合 AI 或 IoT 技術。

### Georepublic Japan
講者 Leo Watana 在 18 歲前都想當漫畫家，後來在消防隊工作，成為小隊長，也曾打過拳擊。從拳擊引退之後，Leo 決定成為工程師，2015 年在夏日學員開始學寫程式、參加 hackthon 2018，也參加 code for all 的 summit。他覺得程式碼是個很有創意又很有效率的工具，想用現代科技解決問題好幫助人，所以就從 Tokyo Fire department 到 Georepublic Japan、又成為了 My City Report 的專案經理，希望可以結合自己的經驗、知識與科技的力量。

Georepublic Japan 的原型是讓使用者拍照， app 會把照片送到資料庫，幫助機器以 Image Classify AI 學習分辨有火災或沒火災的照片。另有像是「路平小幫手」的功能，可回報路上壞掉的地方，讓市民一同維護市容，例如前些日子千葉颱風發生時，居民可用這個 app 回報尚未修繕的東西。

### City report
講者介紹可結合公民與地方政府的開放協作工具，以及「Found, Share, Solve together」的協作過程。專案自治體的資訊是跟地方政府連結，例如千葉市的公務員可以看到和歌山、加賀、千葉、花卷、東廣島市、大津市等等城市加入這個專案自治體。

### MCR for Road Manager
MCR 透過由民眾提供與分享道路障礙的照片，回報需要修繕的道路，也能追蹤障礙排除進度。使用者只要在開車時將智慧型手機放在車上，MCR 便會計算、找出路上不平的地方，再送進資料庫裡給 Construction Bureau Project of Tokyo 應用。MCR 幫助民眾不要被路上的洞困住，千葉市長曾為此發了一篇 FB 貼文，認為公民跟政府一起解決公共議題。在與政府的合作上，政府可用訂閱制的方式使用 MCR，比政府自己開發 app 成本還低。

### 科技改善地方觀光體驗與災後重建
此議程共介紹了以下公民科技的應用方式。
使用 AR 技術做 Reak world 祭典，包含虛擬實境體驗，內部探險、歷史探險等等。
其他自治體推出旅遊書、畢業旅行、觀光情報服務、觀光客、餐廳服務。
港、製鐵所、自行車工廠等地方希望在 2020 推出一條龍服務，打造和現不同的旅行網站。
一級產業、二級產業、三級產業、自然或文化等各類別的體驗類觀光服務興起，例如農業體驗。
旅遊 app 讓地方政府可以好好維護。活動會自動抓取和格式化各個地方政府的公開數據，例如避難所是政府公開發佈的資訊，所以可以自由使用。使用旅遊 app 時會有一些公布在地圖上的資訊，現在開發的產品會顯示發生事件的地點時間、並允許你查看事件，幫助旅客取得旅遊資訊。義大利就有三家公司介紹了這些旅遊資訊，日本正在示範給世界看日本人可以怎麼做，如果做出了有趣的模型，就可以轉換運用到國際上，讓每個人都想要來東京玩。

### 經產省數位轉型計畫
在一場討論與政府合作敏捷開發 (agile software development) 的計畫中，政府系統 IT 承包廠商談及敏捷開發的重點是參與式設計與儘早獲得使用者反饋。私下與發包的政府單位聊天時，他們認為敏捷開發很好，可以讓政府單位不用等到計畫結束再驗收，中間即可反饋。但他們的計劃不像台灣一般軟體專案會有期中驗收點，專案最後才驗收。因為政府原有的公務員不知如何開出符合科技業業界標準的開發規格，致力於推動數位轉型(METI DX)的經產省(METI) IT Innovation Division ，在過去兩三年間，聘僱了十位來自私部門科技產業的專業人士進政府，協助政府開出標案、科技產品規格與專案管理。

值得一提的是，在會議上遇到積極參與的公務員有來自地方政府，也有中央政府的。一位中央政府事務官表示，地方政府在數位轉型上走得更前面，中央和地方需要多交流，也需要多跟開發者社群合作。從會場使用 360 度攝影機，子母分割畫面、政論節目標題和手卡準備充足的狀況看來，主辦單位準備的十分縝密。

其他相關計畫有：
* gBizID是一年期的計畫，讓公司登記有個統一的 ID 編號與工商憑證，使用該憑證認證政府所有相關的服務。
* jGrants 是兩年期的計畫，在 gBizID 的基礎上，提供中小企業申請政府補助統一的平台。專案發包給大的 IT 承包商，大承包商再轉包給數個小的承包商。但目前進度落後，可能今年底無法如期上線。
* ミラサポ plus  是 6 個月實驗性質的標案，目的為中小企業補助申請的網站 提供一站式服務與升級。可以線上電子化申請。改版前後的對照詳見下圖。
* ミラサポ plus 升級的網站頁面主要分為三個功能：
1. 政府補助接口： IT 顧問公司
1. 補助搜尋：IT 公司 + Code for Japan 
1. 案例搜尋：給群眾外包網站 Topcoder 作為挑戰徵集設計，該公司選 提案供政府作最終決定。



在開發此專案時，遇到以下困境：
* 功能無法順利整合：三個功能分成三個標案發包，由中小企業處的三位公務員分別作為產品負責人，並由 IT 辦公室的一位人員擔任整個產品的專案經理。然而，最後該專案經理沒有時間顧到三個功能，這三個功能的開發也沒有經常開會，因此最後無法順利整合。因此新的一年的標案會交由單一公司承包，產品負責人 (product owner) 為 IT 辦公室的人直接負責。
* 在群眾外包 Topcoder 上的挑戰是一層一層逐步開規。規格算開得滿好了，在設計稿還有提供使用者人物畫像 (Persona)，但有繳交作品的不多，最後由一位印度設計師獲得資格。
* 對於中階事務官而言，推動數位轉型最大的挑戰是說服長官，除了一邊希望能做出代表作說服長官，目前也正著手研擬給高層的數位政府課程，未來也希望能聘僱工程師與設計師進政府，可以自行開發部分服務，加快速度與溝通流暢。

ミラサポ plus 的開發經驗讓參訪代表有以下反思與心得：
* 在思考數位轉型時不是先引入工程師，而是有能力寫出好標案的人和做好科技專案管理的人。
* 接觸到的一些政府中層事務官認為數位轉型很急迫，但是還是比較把工程師或是 IT 人員作為工具人思考，認為只要規格書開好，工程師照做即可，因此可以轉包再轉包，或是開非常短期的標案，甚至考慮要讓公務人員不用學習程式，直接使用 Microsoft PowerApps 開發所需應用。聽起來實作的工程師未有機會參與設計討論，是滿可惜的一件事。
* 政府有意願導入敏捷開發，但是卻在某些專案上切割主頁功能分包給不同廠商，且在敏捷開發的過程中未互相合作，各做各的，導致功能合不起來，體驗不流暢。
* 面對開源社群的態度以生態系去考慮，希望有更多開發者加入。考慮開源的原因是避免被單一廠商壟斷標案，尚未考慮到公眾參與與檢視的部分。

# 臺灣出訪團的議程
這次在 Code for Japan Summit，有兩個介紹臺灣社群及相關專案的議程入選進正式名單，因為兩個分享內容都很多元，因此幾乎相關人員都一同前往日本參與議程報告，最終出訪人數達 8 位。

### 議程一：g0v 國際交流專案分享
g0v 零時政府於今年成立國際交流專案小組 g0v-intl，希望能將在全球深受好評的 g0v 經驗更主動、持續的帶到國際舞台，並且與國際夥伴促成實質合作。這次在大會中分享的，便是 g0v-intl 與日本、韓國社群一起舉辦三國面海黑客松 Facing the Ocean Hackathong 的經驗，緣起於 2018 年 12月 一場首爾見面會決定一起看看公民科技社群的新發展可能性，讓日本、韓國、與台灣一起合作在沖繩辦跨國的面海黑客松，讓來自不同國家、不同領域的人一起交流、因而看見各自國家正在面臨的社會議題，自各怎麼解決，不同文化的人一起協作思考、討論以及共同合作提出解決問題，沖繩面海松活動受邀沖繩當地媒體採訪。

### 議程二：g0v.tw 台灣零時政府經驗分享
g0v.tw 參與者介紹幾個經典的專案，包括查核不實資訊的「Cofacts 真的假的」；促進民眾參與政策討論的「vTaiwan」；與政府協作，優化政府開放資料申請流程「GovDataOpener 資料申請小幫手」 。緣起於 2018 年 12 月一場首爾見面會「面海黑客松」也是介紹重點之一，日本、韓國、與台灣的社群參與者因為想探索公民科技社群新發展可能性，因此日本、韓國、與台灣社群決定合作在沖繩辦跨國的「面海黑客松」，透過與來自不同國家、不同領域和不同文化的人一起協作思考，促使參與者們反思各自國家面臨的社會議題、尋找解法。

### 促進跨語言、跨年齡與跨文化的交流
Code for Japan Summit 2019 的議程內容十分豐富，同時段有高達五場的議程同時進行。主辦方十分注重參與者之間的交流，為了讓不懂日文的參與者也能順利參與，每個場地均設有同步自動翻譯的投影幕。活動的形式也十分多元，有供日本各地 Code for X 聯盟成員呈現靜態成果的攤位區展版及贊助商攤位，部分講題亦有專業人員提供現場繪製圖像記錄並展示於走道，另設有需多促進公民參與的工作坊及動手整理資料的培力課程，以促進參與者互動。晚會上融合各地特產的餐點，也可見主辦方煞費苦心，期望能強調多元地區、多元文化的特色。

再者，主辦方亦十分重視各年齡層公民的參與，為了讓育有子女的會眾安心參與議程，會場設有兒童遊戲室，由專業人士提供托兒服務。Code for Aichi 提供經驗，他們為了讓小孩可以跟大人一起學習程式，還會設立「寫 code 道場」；此次會議設有高齡參與者的議程，真正達到促進各年齡層社群參與者互動的目標，至於性別方面，有會眾提到女生參與者的確較少，希望在議題設定上可以增加一些女性相關的議題以吸引女性、突破目前男性為主的情況。

Code for Japan 是一個多中心社群，由各地區的 Code for X(該市區名字）是獨立於 Code for Japan 而存在的社團法人，有些社群會雇用全職員工處理庶務，有些社群則由當地公務人員熱心貢獻，每個地區的社群運作方式與活躍程度不一。由於地方政府很希望發展當地科技，Code for Japan 會幫忙串企業和科技的資源，跟地方創生有非常緊密的關係。地方政府搶著舉辦 Code for Japan Summit，所以在年會中會有很多地方政府相關的議程，且為了創造不同區域間的參與者交流，舉辦地點刻意輪一年在城市、一年在偏鄉，期望藉此壯大社群規模，也可以接觸更多地方政府。此外，日本公務員參與社群的比例很高，超過一半的年會參與者都是公務員，在會議上分享職涯心路，畢竟一個人在公部門很寂寞，來參加會議可以充電和互相學習。

Code for Japan Summit 十分注重參與程度和互動性，會場配有即時翻譯軟體，並搭配視覺化紀錄討論內容，幫助不會日文的參與者了解議程內容。此次議程不僅內容豐富到分成多軌並進，活動內容也十分多元，有工作坊、程式教學也有玩桌遊。高齡開發者社群頗值得台灣借鏡，可用來教長輩寫程式。

### 整體參與觀察、心得

#### 破除語言隔閡的巧思
Code for Japan Summit 2019 在加強會眾連結的用心，是此次與會最令人印象深刻的事情。與 Code for Japan 的參與者聊天時，得知他們非常鼓勵整個家庭參與活動，參與者不乏國小國中生，在一旁也有托育室，讓帶著小小孩的參加者能安心地參與活動。雖然整個內容幾乎都是以日文為主，但主辦單位致力於降低語言隔閡，讓不會日文的會眾也不會擔心。除了備有搭配翻譯的 App，還在有外國人在的議程安排工作人員隨時在旁以確保外國人能能跟上狀況，彌補日英即時翻譯無法順利運作的障礙。相較於去年，今年的英文對話比例提升很多，這有賴於會眾積極的參與及互動，例如在看起來像是日本人主場的議程時，難免會擔心自己待著難以參與，該場次的主講會很客氣的來到身旁友善的邀請加入工作坊實作並加開英文桌，也同時鼓勵日文母語的參加者盡量使用英文，讓我們能以舒適自在的模式參與活動。透過這樣產生的連結性是非常強的，短暫的相處確可以馬上成為朋友。

#### 公私協力、跨領域合作
這裡和台灣公民科技社群不一樣的地方在於參與者很多都是公務員，有地方有中央，在地政府也用很多資源在支持這些在地專案的運作；台灣則多為由民間社群發起，邀請政府組織一起來參與，非常不同。在日本有很多的專案是政府與企業一起合作，對於認識日本政府與企業之間協作如何運作是台灣很好的參考，無論政府組織或企業內部在運作時勢必會遇到許多挑戰，例如：組織文化、多方利益關係人、跨領域協作、跨部會溝通等等。在這樣的場域裡一般人可以聽到很多來自不同政府單位、企業、不同領域的專業者的聲音，彼此能建立對話、了解彼此。「共同找出更好的溝通方式」這是來自於講者分享與政府合作時覺得很重要的事情。專案則同樣都由「人」出發，致力於解決「人」的問題，如果能整合更多資源，大家一起進來把事情做好，可以讓更多一般人能受惠。在活動中聽到很多講者在做專案的初衷都是來自於，希望能「改變」、「解決問題」，沒有人做那就自己來吧！這樣單純的初心令人很感動。

#### 深化人際連結
此次最大的收獲在於締結人與人之間的連結，透過 Summit 活動能將人聚集起來，帶入到不同的城市，不同城市間的人有機會認識彼此、體驗彼此的經驗。國際之間公民科技社群的連結似乎也是在這之間逐漸建立的，透過創意與實作改變社會，如果覺得只有少數的人在努力的話，偶爾也會有點懷疑「這樣的方向是正確的嗎」、「會不會有其他的可能性」，可是把連結擴大之後，就會慢慢發現原來世界上的許多地方，都有人在做著類似的事情，大家的目標和願景都是類似的，這些能量其實都可以持續累積，慢慢壯大成新的力量。從路平專案、或是群眾協作的地圖回報系統，可以看到每個人的貢獻與付出，都會為另外一個人提供下一次貢獻的舞台與機會。資訊的分享與合作，最後能夠積累出非常大的資料量，而這些珍貴的資訊，就能幫助其他一起在這裡生活的居民，免於危險、得到政策上的便利，這些背後的價值都是無價的。
