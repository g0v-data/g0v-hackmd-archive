### [線上課程與影片] g0v 帶你來提案
- 講者：Tiff
- 影片：https://youtu.be/K_Tj1nx4lF8?t=13477
- 簡報：https://docs.google.com/presentation/d/1_ORgz-cVMMlVCQeP3bl2fQ_T1NS833fyEZOivMXG5dU/edit
- 探討順序
    - 你最想挖哪個坑？最有情緒、最有畫面的那個，從這個坑開始探討吧！
        - 1.What? 議題現況為何？
            - 用陳述句列出已達到與待解決的現況，記得放上主詞
        - 2.Why? 造成此議題現況的重要原因
        - 3.So what? 造成的影響
        - 4.哪些有實證，哪些只是假設？
        - 5.你最想解決「誰」的「什麼困難」？
            - 核心問題與表層問題都需要有人解決
            - 「哪些問題相對沒有人解決／還可以做得更好？」（有人做但是為什麼問題還在？）
            - 「我對哪些人及問題較有把握？」（對人及問題的掌握度會影響心力與時間）
            - 「哪些人改變後最有機會影響其他人？」（借力使力）
    - 現有哪些解方 -> 設計原則與限制條件 -> 提出解方
        - 1.目前有哪些解決方式，其優劣勢？
        - 2.星河：設計原則與限制條件
        - 3.依循星河列出各種解方（MVP）

### [線上課程與影片] 我們與專案的距離
- 講者：Peter 
- 影片：https://youtu.be/OlpBj6pzH4w
- 簡報：https://g0v.hackmd.io/dF4YjaJeS0esamJnCiXIww?view#%E6%88%91%E5%80%91%E8%88%87%E5%B0%88%E6%A1%88%E7%9A%84%E8%B7%9D%E9%9B%A2---Peter
- 提案引導欄位的表單：https://forms.gle/wxk5nvRYGwmMod5t8
    - 你想開啟的專案名稱是？
    - 你想要利用這個專案達成什麼目標，或解決什麼問題？
    - 你在這個專案會希望用什麼工具作為紀錄軟體？
    - 你在這個專案會用什麼工具作為會議軟體？
    - 你在這個專案會用什麼工具作為聯繫軟體？
    - 你在這個專案會需要收集什麼資訊？
    - 你在這個專案會想要讓誰看到或使用你產出的結果？
    - 你預計每週在這個專案上投入多少的時間？(這個部分的長短不會有任何影響？)
    - 你想要找什麼樣的人加入這個專案的團隊？
    - 你想要透過什麼樣的方式找到新的夥伴？
        - 人際連帶、社群連結、公布資訊、其他
    - 支持你開啟並繼續投入這個專案的原因是什麼？
    - 請用一句話介紹你專案的目標或者是效果。


### [提案模板] 提供政大資科系所軟體工程概論課程，專案構想模板
- 由 Isacl 簡化自 專案介紹空白模版
- 模板網址：https://g0v.hackmd.io/vTlggVuHQz6H--UwyfiFCg
- 欄位：
    - 直指問題
        - 想解決的問題是什麼
    - 資料在哪裡
        - 有現成資料、平台解決這樣的問題？若有，資料在哪裡？ 請附上資料 url 
    - 開發資源
        - 請寫上 source code 存放的位置（version control repository location），以及 Issue Tracking System 的位置。
        - 如有團隊專屬聊天頻道，請寫上頻道在哪裡，如何加入。
    - 畫面或流程
        - 是否有簡單的畫面雛型或是流程代表你想做的事情？
    - 授權
        - 如果這是一個新專案而非既有專案，想要用什麼授權方式釋出你們的成果？
    - 目前開發進度、狀況、遇到的問題


### [提案模板] 專案介紹空白模版
- 模板網址：https://g0v.hackmd.io/cWAH7w8XQl6zQqzvptTTcw
- 欄位
    - 專案簡介
        - 緣由介紹，為什麼會想發起這個專案，一些背景資訊
    - 要解決的問題
    - 預定使用者
        - 成品要給誰用、在什麼場合用、怎麼用
    - 預定功能
        - 成品要有哪些功能，來滿足上述使用情境
    - 現有類似專案
        - 現成的是否可以直接使用？或者有什麼不足之處？國外專案可參考？
    - 相關專案
        - 是否衍生自某專案 / 是否衍生出某專案 / 是否 API 串接自某專案
    - 授權方式
        - 程式碼，如 MIT/BSD
        - 文件部分，如 CC-BY
    - 使用資料
        - 會使用到哪些資料來源、各是什麼授權
    - 專案目前狀態
        - 構想 / 規劃 / 雛形 / 實作
    - 利益揭露
        - 牽涉到哪些組織團體、有哪些已知的或潛在的金錢或物質或無形利益報酬

### [提案模板] 公民科技獎助金 提案要求格式
- https://grants.g0v.tw/
- 欄位：
    - 請以簡短且非專業人士也能理解的方式介紹此專案。
    - 提案者簡介。
    - 請簡述提案者在專案中的角色（例如：專案經理、全端工程師、顧問等）、專案成員與團隊背景介紹。（團隊現在有什麼人才？還需要什麼人才？）
    - 你為什麼要做這個計劃（個人動機）？
    - 這是一個現有的計畫嗎？若是，請說明是否為開源及提供網址
    - 本專案是否有非開源元件？
    - 請以此句型釐清專案目標、受眾與獨特性
        - 「為了解決 ______ 遇到的 ______ 困難，我的專案提供 ______，可以做到 ______ 。我的專案跟 ______ 不同，因此具有不可取代性。」
    - 這個計畫要解決什麼問題？
        - 200 ~ 500 字
    - 你預計用什麼方式解決此問題？
        - 200 ~ 500 字
    - 為什麼你的方式可以解決此問題？
        - 200 ~ 500 字
    - 請列出這個計畫的目標對象，他們的需求、情境與使用動機
        - 200 ~ 500 字
    - 請說明你如何讓目標用戶知道這個服務並完成其第一次使用，以及在他有需求時願意再來用這個服務？
    - 這個計畫正在／預計跟什麼團體合作？
    - 請提供本計劃的示意圖或介面設計草圖（請以連結方式提供圖片）
    - 是否已有相似的專案，本計劃的差異與不可取代性為何？
    - 過去作過什麼開源開發計畫（OPEN SOURCE PROJECT）？（可提供 GITHUB 帳號）
    - 請說明本專案會產出的開源軟體套件或開放授權文件
        - 請條列個別元件的輸入輸出或其功能
    - 預計八個月內將花多少小時作這件事？
    - 打算如何讓社群參與以及回饋意見？
    - 你覺得這個提案可能會面臨的困難有哪些，你又會如何解決這些困難？
    - 計畫結束後未來可能進一步的發展？
    - 若有專案介紹的共筆或是投影片，請提供

### [提案模板] 零時小學校 提案要求格式
- https://sch001.g0v.tw/
- 欄位
    - 專案簡介
        - 請以簡短（150 字內）且非專業人士也能理解的方式介紹此專案。
    - 請提供提案的縮圖
        - 縮圖有助於讓您的提案被更多人看見
    - 提案團隊名稱
    - 提案團隊簡介
        - 請簡單介紹每位成員，包括背景、在專案中的角色（例如：專案經理、全端工程師、顧問等）若還需要徵求成員，也請在此項說明（還需要什麼人才）。
    - 為什麼要做這個專案？（動機）
    - 請以此句型釐清專案目標、受眾與獨特性
        - 「為了解決（某使用者族群）遇到的（要解決的問題）困難，我的專案提供（某種解決方式），可以做到（如何解決問題）。我的專案跟（現有的解決方式），在（差異與區別部分）不同，因此具有不可取代性。」
    - 請列出這個計畫的目標對象，他們的需求、情境與使用動機？
    - 請提供本專案的示意圖或介面設計草圖問題的標題
        - 請以連結方式提供圖片。
    - 你覺得本專案可能會面臨的困難有哪些，你又會如何解決這些困難？
    - 本專案結束後，未來可能進一步的發展？
    - 若有本專案介紹的投影片，請提供。
        - 請以連結方式提供。
    - 你有什麼其他想補充的事情嗎？
        - 建議題目：這個專案正在／預計跟什麼社群／團體合作？
        - 建議題目：你的專案有面向某個社群嗎？要如何讓社群和使用者參與專案，並回饋意見？
