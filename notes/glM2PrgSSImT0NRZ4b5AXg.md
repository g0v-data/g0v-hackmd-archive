# g0v Summit 2024 組織架構

## 前提

Summit 工作小組的目標：

* 要能夠完成活動：活動有辦出來、沒有開天窗、沒有掉太多球。
* 要能發揮品牌影響力：主要是能與不同的群眾對話、傳達 g0v 精神、邀請協作。
* 要能執行「治理」的工作
* 加分：減少大家開會的時間
* 加分：降低不確定性，減少壓力

### 能夠完成活動

活動有辦出來、沒有開天窗、沒有掉太多球。

### 能發揮品牌影響力

主要是能與不同的群眾對話、傳達 g0v 精神、協作。

  - 不同群眾：以下是對話切入的角度，而不是固定的分類。角度可以多重，可以重疊，可以變動。例如往往，廠商也是社群的一員，甚至也提供贊助。
    - 社群：因為對議題、主題關心而聚在一起的人。
      - g0v + 黑客 + 公民科技
      - 開源 + 技術社群
      - NGO
      - 學術研究
      - 公部門 + 法人 + 智庫
      - 新創 + 商業
      - 台灣 vs 港日韓東南亞 vs 國際：以上的每個角度都有在地（台灣）、區域（亞太）、國際（亞美歐非澳）的不同脈絡。
    - 會眾：拿票進場的人、線上參與的人
    - 講者：付出時間精力，一起經營活動內容的人
    - 贊助商：物質上支持活動舉辦的人
    - 廠商：接受委託，完成一部分工作的人
    - 一般大眾：透過媒體、社交平台得知相關訊息的人

### 能執行「治理」的工作

> [name=pm5] 這些事過去多交給總召決定。

  - 必要時對外代表工作小組
    - 向 OCF 送出申請
    - 場地簽約
    - 廠商簽約
    - 發聲明
    - 發感謝文
  - 保管所有個資與機密資料
    - 誰決定哪些資料是機密？
    - 可能是機密的資料
      - 眾人個資
      - 廠商報價
      - 贊助者，必須以贊助者同意的方式露出
  - 執行 Code of Conduct
    - 必要時取消違反者與會資格
  - 請款
    - 誰可以請款？
    - 誰決定一筆款項能不能支出？
  - 誰決定誰應該調整工作內容、離開工作小組、誰要 take over？
  - 任何無法有共識的事情，誰下 final decision？
    - 沒有決定，會無法做事情。
  - 沒有明確 owner 的問題，誰下 final decision？
    - 有颱風地震戰爭，活動要取消嗎？

## 提案

### 提案一：總籌 + 組長 + 顧問 + 工人

> [name=pm5] 原本我想提案能不能有個 core team 或 committee + work group 形式的組織，就可以辦完 Summit。但看了一下所有要完成的工作內容，覺得這想法好像不太實際。所以還是先提一個基於過去的組織架構，另外增加一個「顧問」來補上之前缺乏的部分。
> 
> 有幾個組還是需要組長，是因為，他需要很多工人：場務、議程、設備、紀錄。有些組比較可以不設組長，自行協調，有人來參加跨組協調就好了。

- 工人
    - 例如：場務、議程、設備、紀錄組，需要超過 8 人的人力
    - 接受組長排班、安排工作內容
- 顧問
    - 提供外部觀點，協助 Summit 團隊瞭解情況、釐清問題、檢討方案
    - 緊急與重大事件的商討對象
    - 由總召參考各方建議組成的 shortlist 名單
        - 3~5 人？
        - 幾個建議的方向（不用全數符合）：
            - 過去幾屆的總召
            - 擔任過工人
            - 參與過社群
            - 保持多樣性
            - 有來自不同規模大小的專案貢獻者
    - 與總籌定期討論
    - 希望是提供外部觀點，所以盡量不要同時是負責執行的人。
- 組長
    - 負責各組事務
    - 要出席跨組協調的會議
    - 要列預算
    - 可以請款
    - 會收到出款通知
- 總籌 = 總籌組的工人 + 總召
    - 總召曾經有 1 人、1 人 + 幾位副召、1~3 人共同總召，等等方式。
    - 過去幾乎所有「治理」的工作都由總召決定。
        - 當然都會徵詢工作小組的意見。
        - 今年也可以討論有哪些「治理」的工作，可以先訂出流程，或是可以由組長們的共識決定。
    - 也可能再加進贊助 + 財務

以上所有人都：

- 可以用 `summit2024` Hackmd team 開共筆
- 視情況會安排活動期間食宿
- 會拿到紀念品
- 可以出席慶功宴
