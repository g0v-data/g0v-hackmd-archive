## 提案優化工作坊回饋：

也有製作行人行走風險指數地圖 https://commusidewalk-app.vercel.app/

## 諮詢先輩 1：superbil


### 諮詢時段： 1/30 20:00~21:00


### 請描述想諮詢的問題：
> 請盡可能包含困難點的情境描述、已嘗試過的方法但不可行的原因、希望能獲得何種協助。

1. 前端使用 React Native或 Flutter來實現跨雙平台 (IOS & Android) 的APP。或者使用非APP 的其他解方:
2. 後端會 建議使用何種語言、框架?  
3. 或者有其他簡單的工具平台: firebase，雲端的 Serverless 方案比較推薦
4. 有無較快速的方式? 可先產生 MVP 產品，開始驗證效果。
5. 圖資來源: Google Map 、ArcGis 等平台的 API 呼叫，有推薦的嗎?
6. 一些發文功能，如上傳照片等等，是否會對後台頻寬處理壓力過大、或是資料量大時，對 儲存空間、資料庫處理的 負擔過重等等。 牽涉到 我們 APP 是否要設計 提供使用者上傳相片等功能，或者免費使用，設定使用上限。





### 諮詢紀錄
1. Line 機器人、嵌在 Line 中的小程式( Web APP)，考慮台灣人 使用 Line 較多，故以 Line 為方向。
Line 記錄功能，做不到，但能提供上傳、下載路線等 互動功能。
地圖功能只能用原生的 API 呼叫，可以先做 Android 或 IOS 其中之一，再刻另一個平台。
2. python: Django 。postgreSQL、postGIS
https://www.django-rest-framework.org/
記得容器化，學一下 Docker
3. 建議不要 serverless (fass)
4. MapBox OSM 的資料，可以取得 & 回傳
5. IOS: api
影像、圖片檔案 丟 CDN。(不建議)
6. 建議找 Yellow Soar，有相關後端經驗
 g0v slack  @yellowsoar
