# 新聞啄木鳥 評價系統1.0

> [點此觀看原始內容](https://g0v.hackpad.tw/npnPyM9z40Q)


## 設計理念

APP4AM除黴的核心價值是建立一個新聞的評價平台, 聚集眾人之智慧來讓普羅大眾了解新聞的問題, 免除錯誤資訊的誤導及抵制壞新聞. 評價機制在使用者方面, 有分層級制度 - 一般使用者, 認證使用者, 專家使用者. 目的是為了讓新聞評價能夠更有專業性與可信度. 這樣的機制下會有權重與門檻的設計困難, 因此在5次除黴會議後, 決議先以建立最基礎架構, 後面的權重和門檻意義就讓評價平台來自我形成

### 關於此版本

除黴計畫是個無法做短期的計畫，但在資源不足的起始狀態下，我們不會好高騖遠，紮實的打下基礎才能考慮未來。

因此計畫必定會分成幾個不同的階段，**新聞啄木鳥1.0**是基於短期目標所開發的版本，目的是求開拓使用市場，關鍵的數據分析及專業性在此版本仍處於開發測試接段。

### 評價系統

有感於目前新聞監督機制的高門檻及微乎其微的影響力，我們著手構思一套夠具潛力的評價系統，來嘗試從更高層次的角度來改善現有機制的不足。
    - [x] 以明確的違規項目取代過於主觀的數字式評分
    - [x] 降低專家參與門檻，讓專家意見成為新聞外的補充資訊
註：在沒有充足開發資源的情況下，一開始無法使用複雜的運算來達到高信度的調查數據，
    僅能先以此方向，吸引使用者逐漸習慣評價新聞的動作，未來再加強評價的精準度。

### 議題分類系統

現有新聞訊息越來越片段化(即使是第一線媒體)，也難以抵抗UGC(使用者參與產生內容)的趨勢，因此需要更有層次的新聞分類架構，特色如下：
    - [ ] 以議題為分類架構，使得資訊破碎的新聞可以在同一議題目錄下集中，讓使用者較容易窺知完整真相
    - [ ] 保留傳統分類標籤(ex: 政治、社會、娛樂等) 滿足無法習慣議題導向瀏覽模式的使用者，也作為搜尋新聞時基礎過濾標籤
