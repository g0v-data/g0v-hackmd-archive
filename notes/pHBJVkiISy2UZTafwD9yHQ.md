- 吳老師問為什麼用 PostgreSQL 不用 MySQL
    - Johnny: 因為看到 MySQL 被 Oracle 買走，對於之後的發展有點concern
    - Ronny: (雖然現場我沒說，不過我可以依我在業界的心得在 Hackpad 分享一下心得) 商業使用上，如果要求穩定度， MySQL 比較好，因為他的 replication 機制非常健全，也有很多追求效能極致的方式，但是穩定度相對不重要的服務，PostgreSQL 的好處是 9.3 後支援˙了 JSON type ，因此可以把 PostgreSQL 當作 NoSQL 使用非常方便，而且 PostgreSQL 搭配上 PostGIS 後在 GIS 超強超好用(MySQL 在 GIS 上超級弱) ，所以 startup 或是想速速開發出服務丟出來的話，PostgreSQL 比較適合。
