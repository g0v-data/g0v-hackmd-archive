# G0V UI

> [點此觀看原始內容](https://g0v.hackpad.tw/Mb2yoaVY378)

成果 ： [http://g0v.github.io/ui/](http://g0v.github.io/ui/)
Why 為什麼要這麼做 ：
- 未來任何人建立新專案時，可以來這裡直接參考、使用這裏的所有 UI ！
How 怎麼做：
- 找出通用的主題架構（eg. 官網首頁、救災型、會議型、列表資料型），讓設計師跳坑著手設計 UI
- 設計師投稿到 [https://github.com/g0v/ui/](https://github.com/g0v/ui/issues)[issues](https://github.com/g0v/ui/issues)
    - 必備元素：一張示意圖 & Sketch 檔案
-  工程師拿到示意圖與 Sketch，完成  component 製作
What 需要什麼：
- 首頁 Landing Page 設計 (Michelle cc0)
- 設計師投稿教學視覺化設計 (Eric)
- 宇廷建制網頁
- 設計師來投稿 UI！
Who 誰要來使用這個網頁？用樂高來比喻！
- 設計師：設計 UI
- 工程師：製作成可用元件 component
- 訪客：來用元件吧！用 Component 疊出自己要的網站！

設計師填坑方式：

1.  從過去的專案找出  UI
    1.  尋找以前的Sketch
    2.  看已完成的專案, 重新繪製Sketch
2.  新的專案先做UI
