### Cofacts
* Hackmd 作為討論紀錄 work journal、technical design doc, research doc 的 knowledge base，以 hackmd book mode 彙整在一起
* 確定要開發的 item 會在對應的 github repo 開票，進 code 後 resolve
* 帶狀的專案，在 breakdown 成 github 票之後，還會在 github organization 開 project 管理專案下的票的進度
* g0v slack 聊天、交換進度、接 github 進來讓人進去 code review（非程式開發者亦可透過開發者留下的 screenshot 確認 UX 符合預期、並得知程式方的最新進展）
* 對外佈達使用 hackfoldr (因為當時 hackmd 還沒有 bookmode 或是 bookmode 還不能自訂 URL；應可以 hackmd book mode 取代之)；對外聯繫使用 Google groups (collaborative inbox 模式) 管理採訪邀約之類
* 非公開事項資訊交換，使用前述 google groups 或 Facebook messenger
