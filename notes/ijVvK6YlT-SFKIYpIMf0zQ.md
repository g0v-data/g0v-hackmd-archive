### #1 可以如何妥善達到團隊與團隊成員在對外事務上的平衡，像是有些學員透過團隊內成員私人訊息聯絡詢問相關資訊，團隊成員使用團隊信箱寄送信件時是以個人名義署名等

::: info
Peter: 
跟團隊成員討論這件事其實蠻重要的，尤其是專案的規模從一人走向團隊的時候。在一人的時候，專案的發起人就相當於專案本身，但如果團隊逐漸擴張，這件事就有討論必要。我建議討論的內容可以從以下開始：
1. 團隊信件的署名格式（窗口姓名、團隊代表的形式）
2. 內部討論的議題：團隊的就在團隊，個人的決定與結論最好不要直接成為團隊的結論！
3. 治理的問題一直都是有趣的事情，多多討論是可以先行遵守的一個基本原則。

治理的事情遇到衝突，很多時候其實不是惡意，只是為了專案好。有明確主導者效率很快，但當領導者無法扮演好角色，團隊就會出問題。唯一個共同規則是有事想做就丟在團隊群組討論，前期投資時間讓大家討論出一些共識，這些投資的回報會在後續回收

Isabel：
治理是專案比較深的問題。對外是代表性的問題，兩個問題：誰可代表專案、誰可以代表 g0v。治理有幾個重點，許多新參者會問怎麼做決定，目前有提一個概念「坑主原則」，有些專案有坑主，有些沒有明確坑主。

Teemo：
推薦《社群藝術（第二版）》：如何管理與團隊成員一起做開源專案。要放手一些事情讓團隊成員可以參與，因此可以先討論好誰是負責對外的公關，重點是分組分工要先講好，會遇到的情境可以先討論。要時常回來討論代表性問題、角色分工、事情主責者。
:::

### #2 我想問當專案遇到瓶頸，無法突破，又一堆挫折、一整天沒進度，要怎麼辦
:::    info  
Peter：
把排序排好：你要先顧好你自己->身邊的人->事情
如果發現你遇到某些挫折，不僅是專案，也許是生活上的，在這樣的情況下要讓自己回復到比較好的狀態。如果專案沒有急迫性的問題，可以先放下專案，好好照顧自己，讓自己先好再回頭做專案。有時候挫折感是來自己的狀態，而不是專案本身。可以培養立即可以消化情緒的方法，我稱為逃生口或山洞。

Teemo：
如果是寫程式上面的遇到的問題，卡住 15 分鐘內就要問人。比較大的問題，例如帶 side project 專案，我覺得要多去找其他的人聊聊，最好是找不同領域的人，和他分享你的現況，請教對方的建議，通常卡住有時候是知識不夠、啟發不夠、機運不夠，透過與其他人聊聊，也許會有突發奇想有些幫助。
:::
### #3 好像很多社團之類的禁止宣傳，還有哪裡可以找人?(特別是有特定專長的)
:::info

Peter：
- 可以直接找常在社團中發言及互動的人。如果能夠直接與人建立連結，會比直接在社團宣傳更好。台大最近有弄了一個 side project 網站 dzoo（目前限制台大生使用）。
- 去參加活動跟找人討論也是建立連結的好的方式～～～
- 找到有興趣的人是第一步，如何讓這個人進入並參與專案是第二步，如果降低大家加入的門檻，也可以讓大家更願意進來！

Teemo:
- 去各個大型的研討會，例如：SITCON、COSCUP、PyCON
- 開會前先把共筆開出來，並先寫下 rundown，讓大家知道今天要討論什麼，若不能參與，也可以去看會議紀錄

Isabel：
- 找人是一回事，專案執行模式會決定你好不好找人，最好找人的是開源模式。在 g0v 做專案的優勢是這邊人很多。讓每次會議都有會議紀錄、成果有開源，這樣就是大家路過就可以掉進坑的狀態。因為當專案封閉起來，新參與者就要不斷回顧以前做什麼。如果專案是 side project 更需要這樣做，需要花時間和心力把共筆等資訊建立起來。
- 沒有 agenda 就不值得開會，因為大家都很忙，所以都會主動建議「討論前先開共筆」。把 agenda 從一開始的會議通知變成會議紀錄。記得要有 action item，用上次的 action item 開頭與收尾。
- 有些專案會用 Trello 追蹤。

:::
