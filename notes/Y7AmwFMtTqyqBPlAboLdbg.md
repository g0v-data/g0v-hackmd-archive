# 客戶交易智慧評級系統
- 出面：TC
- 主要任務：檢索大量客戶資料，為客戶提供信用評級
- 數據來源：政府公開數據如公司等級、新聞、專利、判決等
- 預計解法：ML+RL
- 建立數據集 ![](https://s3-ap-northeast-1.amazonaws.com/g0v-hackmd-images/uploads/upload_c27aa4aa8f9c44b259fc25d15560e6dd.png)
- 訓練
    - 公司登記、專利、新聞等都當成二值化flag，投入模型，預測標籤（應該是要多標籤），並列出決定此標籤的因素。
- 實證目的：
    - 以AI工具密切關注客戶動態，遇有變動即時評估，並賦予評級分數，並即時啓動界面，提醒相關人員關注，並採取適當行動。 
- 面臨痛點
    - 面臨痛點（請列點說明）：
        - 企業生存來自於正常交易，然而商業環境瞬息萬變，因此需要透過大量資訊深入瞭解客戶動向、預測可能變化，儘可能降低風險。
        - 公司內各部門均對客戶現況及變動資訊有強烈需求，但未必各部門均能掌握相關訊息，也因此產生內部訊息落差。此一問題甚至可能造成管理缺口，令公司陷入高度不確定風險。
        - 一般而言，企業多以財務或業務部門負責關注客戶現況，但在客戶衆多且訊息紛雜狀況下，個別人員難以發現問題，或無法形成穩定方法論，或彼此溝通，因而使得企業處於被動局面。
- 需求情境：
    - 客戶經營狀況出現變動，經常是交易風險發生的前兆，但限於人力，難以第一時間發現，企業也因此陷入不確定風險。此時需有一助手即時關注涉及客戶的各種公開訊息，例如股價變動狀況、公司登記狀況、法律糾紛等。
    - 日常管理工作進行時，各部門需要對客戶狀況及其利害關係人彼此聯繫關係進行分析。限於時間或人力常不能獲得相關訊息，如掌握不及可能陷入風險，投入專人，成本不堪負荷。需要一可關注外部資訊變動且成本可控之助手。
- 驗收門檻
    - 門檻一：需能對客戶進行密切關注，驗證客戶數為50家（企業客戶）
    - 門檻二：關注內容包括公司登記變動狀況、司法判決、專利申請授予狀況、不少於一個月內新聞。
    - 門檻三：友善的使用者介面：輸入部分名稱、統一編號，即能進行比對並送出建議名單。
    - 門檻四：能於五分鐘內完成診斷型數據分析，快速反應使用者分析需求，並給出相應評級。

# 產業景氣虛擬企業決策支持系統
- 出面：Mike
- 主要任務：收集外部資訊作爲內部高層決策需求
- 數據來源：政府公開數據如公司等級、新聞、專利、判決等
- 導入產業/場域與面臨痛點：
    - AI出現使得商業預測功能變得準確許多，紙業榮枯營損受國際經濟局勢影響極大，又與潮流密切相關，如COVID造成美國紙漿大漲，國內廢紙進口加工成本激烈上升，導致紙業大幅虧損，以2019年爲例，市值蒸發超過400億。但現有各廠ERP系統均無相關模組，也無相關工具進行準確預測，對此，需有針對紙業的銷售及成本預測，並導入虛擬企業，進行預測，並允許使用者以產業虛擬企業進行決策模擬，期待以此案改變紙業經營基本生態。
- 導入產業/場域 : 
    - 紙業
- 面臨痛點（請列點說明）：
    - 紙業原料來源來自於原生料與回收料，前者與國際經濟形勢關係密切，後者則直接連接到ESG議題。由於紙業與消費者密切相關，因此，相關變動將直接影響消費者效益，例如在疫情期間國際紙漿原料價格高漲，直接導致消費者負擔，但在此同時，紙業也陷入嚴重虧損。
    - 經濟數據高度變動下，各時期各特徵影響力均處於變動狀態，傳統函數做法難以長期應用，也不容易獲得明確結論，決策過程反而更加複雜，企業難以因應變動局面，成本因而不下，需有一較能描述複雜變動環境之決策支持系統。

- 實證目的：
    - 基於神經網路建立產業景氣虛擬企業模型，針對就內部需求進行經營決策建議
- 需求情境：
    - 隨時收集產業新聞及財經訊息，並自動進行評價
    - 各種不同基礎的評價作爲特徵進入深度神經網路進行運算，並對未來可能變動情形進行預測。
    - 預測結果可用以支持企業決策（如擴廠擴線、原料採購、未來市場預測等）
    - 投入變動因素（市場、價格等），並據以產生報告。
- 實證題目
    - 以外部新聞及產業訊息作爲特徵，投入基於特定任務訓練的深度神經網路，並就該特定任務可能走向進行預測，並結合LLM工具，對整體狀況提供描述，並就指定規則（利潤率優先、稼動率優先或庫存優化優先等指標）進行建議，作爲決策支持。
- AI算法及任務
    - 運用爬蟲採集資料建構（紙業）產業新聞資料庫
    - 以資料庫建立產業內企業關係圖。
    - 運用新聞資料及深度學習模型驗證各事件正負影響程度
    - 運用模型對新事件進行虛擬企業空間評價，並調整
    - 投入事件後，以ChatGPT等模組對模型整體變動狀況進行描述/對可能發展狀況進行模擬，並以Chatgpt等模組進行預測，並提出經營決策建議
- 期末驗證門檻
    - 門檻一：外部訊息評價多指標排序正確性（90%）,設定多個指標，如船運新聞導致原料價格上升等。
    - 門檻二：描述正確率>85%(描述符合專業經理人一般判斷，且無明顯矛盾為得分，產出一定數量報告，並由多位專業經理人評價。)
    - 門檻三：友善的使用者介面，可自動收集新聞，並定期自動投入模型進行計算，並自動產出報告。其包含分析報表，並以適合使用者閱讀的圖形化展示。
    - 門檻四：每日分析，快速反應使用者分析需求。


- 預計解法：DL+ChatGPT


- 基本設計
    - 以新聞為脈絡，建立相關公司及新聞網路關係圖
    - 引用Elmo系統判別新聞及訊息，建立新聞flag
    - 對同時間紙漿價格指數跑多元回歸模型或深度模型，給出整體評級
    - 引用回歸結果透過LLM撰寫報告

    - 門檻五：以測試集進行評級時，正確率需大於99%

# 企業客戶AI盡調系統(AI-CDD)
- 出面：Richard
- 主要任務：檢索大量內外資料，建立客戶評估報告
- 數據來源：政府公開數據如公司等級、新聞、專利、判決等
- 導入產業/場域與面臨痛點：
    - 紙業交易形態為擔保交易，交易對象包括紙器廠、快遞業者等，客戶徵信工作直接影響經營績效。過去均以人力進行評估及撰寫報告，在LLM出現後，此一工作可望在AI支持下，進行更精準判讀，並能以專屬模組，撰寫報告，提供集團內各公司，，產出完整報告
- 實證目的：
    - 以AI工具自動定期針對授信對象進行信用評估
- 需求情境：
    - 自動根據新聞及公開資料變動狀況，修正客戶信用評價、形成報告，並通知利益關係人
- 實證題目
    - 大宗物資生產企業常須對客戶進行授信，以利銷售作業進行，但授信工作需比對包括公開數據的大量資料，以本公司而言，超過**7大數據源**，期望以AI模型進行比對，並產生完整報告，以利評估人員決定是否授信，更進一步期望以其作爲自動授信評估引擎的基礎。
    - ![](https://s3-ap-northeast-1.amazonaws.com/g0v-hackmd-images/uploads/upload_6c47f80366935dcee00fe4f282a1a47c.jpg)

- 期末驗證門檻
    - 運用爬蟲及chatgpt，對特定客戶公開資料、相關新聞、財務報告，分別進行摘要及分析
    - 運用新聞資料庫，以本公司為起點，建立至少二跳關係圖（本公司、客戶、客戶關係人），並以歷史往來資料建立信用關係圖網路。
    - 出現因新聞事件導致風險等級變動狀況時，取用相關新聞、內部財務記錄、信用記錄，運用chatGPT，針對各相關單位，以其觀點撰寫報告，立即通知，提出預警。

- 預計解法：ML+DL+LLM
- 基本設計
    - 同第一案，僅在第一案後，按照評級資料設計LLM模組，撰寫報告。
