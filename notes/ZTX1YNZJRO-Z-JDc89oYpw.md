## 主題意識

分散式公民行動中，如何快速建立「總機—分機」溝通系統，讓現場訊息

### venev 短講

在這次事件中負責後勤
分散式公民行為、快速建立總機分機系統
原始的溝通方式是透過自己原有的 FB page 或 IM channel
設立中繼站成為總機
- 統一格式：地點、數量、連絡方式
- 送入 google doc
- 再發佈到特定的粉絲專頁
最好的點對點溝通方式可能是無線電
- 問題：設備 (?) 數量有限
au: 卡西 hack logbot 以支援 tag filter
設計 cheat sheet 給駐點人員

et負責的部分：
- google drive folder / google docs 權限設定
    目前分成兩組
    - g0v總機+物資+live組
    - g0v線路組
....剛剛才測試確認 google drive folder 設定權限時可以直接新增一個 mailing list (proxy mode) 或一個 contact group (quick add mode)，mailing list 成員即使個人沒有顯示在分享名單中，也還是有權限～我浪費好多時間在調權限 QAQ
