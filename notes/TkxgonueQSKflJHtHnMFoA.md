### 內容規範
- [x] 內容必須與 g0v 社群有關。
- [x] 內容必須遵守 [g0v 行為規範](https://g0v.hackmd.io/COC)。
- [x] 內容及授權必須符合 [g0v 宣言](https://g0v.tw/zh-TW/manifesto.html)。

### 授權規範
- [x] g0v.news 的文章預設授權是 [CC BY-SA 3.0 台灣](https://creativecommons.org/licenses/by-sa/3.0/tw/)。
- [x] 文章中的圖片、素材請明確標註授權條款及出處，並附上連結。

### 排版規範
- [x] 英數字與中文字之間加半形空格。
- [x] 中文標點符號使用全形。
- [x] 不使用全形英數字。
- [x] 引用文字或影像，必須提供原始連結及其授權條款，可使用註腳。

### 投稿
- [x] 草稿必須放在一份 Googld doc 中，並且將存取權限至少設定為「任何直到連結的人都可以評論」，方便 review 及社群協作。
- [x] 稿件作者必須確認稿件符合「內容規範」、「授權規範」、「排版規範」。
- [x] 稿件作者必須到 #news 表示你要投稿，請在訊息中明確表示「我要投稿」
- [ ] 草稿 Google doc 中，請標明以下 metadata 必要資訊
    - [x] 標題
    - [ ] 授權條款
    - [x] 作者顯示名稱列表
    - [ ] 聯絡人：是作者之一，且是 g0v Slack 帳號
    - [x] 是否用個人 Medium 帳號自行上稿；提供 Medium 帳號
    - [x] 顯示圖片可下載的網址
- 草稿 Google doc 中，可標明以下 metadata 選用資訊
    - [x] 摘要
    - [ ] 自訂網址
    - [x] 標籤：最多 5 個

### 審稿

- 作者修改後，reviewer 進一步修改稿件，確認符合「內容規範」及「排版規範」

### 上稿

- 作者自行上稿
- Reviewer 修改格式
