### 第一波問題


#### 問題一、請介紹一下這個網站

這個網站是基於服貿協議行業對照需求所提供的查詢服務，透過公司符合名稱、統編，或營業項目代碼的關鍵字，就可以查詢特定公司或是營業項目是否為兩岸服務貿易協議中台灣對中國大陸的開放承諾所包含，也就是所謂受影響行業。

#### 問題二、這個方案的成立過程，以及對服貿議題的影響（正反皆可）

請參考[簡報](http://www.slideshare.net/nchild/g0v-remix)花絮，最初源自郝明義先生告知此議題，進而與部分 g0v 參與者開會交流，嗣後經由兩岸協議監督聯盟賴中強律師參與 g0v 小松，與現場參與者討論出[行動方案](https://g0v.hackpad.tw/服貿協議行動方案-Mgg49ShtgEM)，當時主要要做的是[服貿受影響地圖](https://g0v.hackpad.tw/-g0v--SUkMbG4IV6v#:h=服務貿易協定受影響的產業地圖(tisa-map))。後因政府遲未提供官方版行業對照表，遂由民間自行整理出一份名為「你被服貿了嗎？」的行業對照表，至於網路服務版的「你被服貿了嗎」，則是於第肆次國民大會黑客松所挖的坑（專案）。
一般人縱可想像服貿協議牽連甚廣，卻因文本與附件的文字艱澀、複雜難懂，而缺乏進一步了解的興趣與動機，透過迅速查詢自己所處的行業是否受影響之結果，是以成本極低的方式建立民眾與此議題之關聯性，並於查詢結果頁以白話方式說明該項目之開放承諾，以降低此議題的關注障礙。

#### 問題三、發起人是誰？具甚麼樣的專業背景與生命歷程？

發起人（坑主）是 [nchild](https://g0v.hackpad.tw/YUZ2ZOUpQqY#nchild)，原坑在[此](https://g0v.hackpad.tw/-g0v--SUkMbG4IV6v#:h=你被服貿了嗎？（服貿協議行業對照工具rutisa）)。跨領域網路工作者、議題懶人包作者及講者，略懂廣告、電信、智財，曾發起[消費者熱線](http://fb.me/Baagic)、[推動網路中立性立法](http://fb,me/NNinLaw)等社群。

#### 問題四、成立的宗旨與目標？

服貿簽署時最大的問題就是說明不清，沒有完整行業對照表，沒有個別行業衝擊評估，沒有對國家經濟表現整體評估，民眾沒有切入點去了解服貿內容，無從同意或反對。
行業對照本應是政府衝擊評估之前就提供，卻遲至簽署半年後方有初稿，這就是「你被服貿了嗎」的發想背景。為了補完政府應做的功課，讓民眾先知道是否受影響，以便進一步評估表態，遂規劃「你被服貿了嗎」，以提供更簡單易懂、迅速的查詢方式為目標。

#### 問題五、是否有受過甚麼樣的阻力或困難？

以資料面來說，最早的時候並沒有要仰賴官方版行業對照表，本想自行進行並驗證，後因政府承諾提供，加以專案延宕，故決定等待官方版（12月底）。
上述延宕主因與專案動能有關，一開始僅有議題關注者參與，工程師們主要在處理服貿受影響地圖，且團隊成員亦有理解服貿之障礙，對於整合地圖和服貿條文有困難，缺乏設計師導致介面呈現無法定案。此外，反黑箱服貿陣營訴求（如朝野協商決定逐條審查等）提前達成，也讓此議題看似不具急迫性，故前幾個月僅緩慢前進，直至第陸次勞動基準黑客松後，方有視覺設計師加入，完成網站模擬圖，大部份的建置完成與3/17~3/30之間（危機感最高點），其後亦有陸續維護更新。
資料整理與歸檔方面，雖然服貿開放之項目（六十餘項）尚可以一至二人之力完成，但期間複雜的多對多關係與對不起來的例外情形不少，常常需要重新設計架構與重新建立編號系統；將政府提供之數份 PDF 檔內容，整理成結構化的關聯式資料庫記錄，也必須由專案貢獻者以人工為之，雖然我們對資料的 integrity 相當重視，但仍然會產生人為誤植錯誤。

#### 問題六、團隊是如何運作的？團隊人數大約有多少？

遠端協作與黑客松（大小松數次）現場協作，黑客鬆參與討論共二、三十位，Facebook 討論訊息串裡有16位，曾對程式碼進行貢獻者共10位（主要貢獻者有四、五位）。
網站專案的進程，是先由 nchild 討論出互動方式、視覺設計師依照互動方式完成網站主視覺後，程式設計方面與 g0v 的白話文翻譯專案《服貿開箱聞》產出之內容合流，整合公司行號資料，使產品在 3/30 上線，再在日後進行勘誤與維護。

#### 問題七、資料從何而來？如何保證資料的可信度？

服貿協議附件、政府行業對照表初稿，及公司行號開放資料。
官方資料可信度多少，網站可信度就至少有多少，且佐以團隊成員逐條審查，理論上更為可信。

#### 問題八、目前達到了甚麼成果？

賺得媒體：超過 2.6 萬個 Facebook 讚或分享（見網站右上）、媒體報導，紙媒含聯合報與蘋果日報。
上線首 2 日突破 6 萬新訪客，累計至今已服務超過 12 萬的新造訪的使用者。

#### 問題九、服貿議題退燒後，網站是否會做轉型或更新

目前原規劃功能尚未全部完成，且初步達成預期目標，故暫不考慮轉型。
會陸續依照規劃與[使用者回饋](https://g0v.hackpad.tw/你被服貿了嗎-beta-測試討論-2spPf82j4ZH)進行維護、更新，進度可於[此](https://github.com/g0v/tisa-map)關注。



### 第二波提問


#### 1.如何將各行業與服貿條文做連結？

由於服貿開放項目是採聯合國 CPC 分類，需要行業對照表，先轉換為國內主計處的行業標準分類，然後再轉換為公司行號的營業項目代碼。
公司登記時僅有營業項目代碼，對照表連結好 CPC 與營業項目代碼後，可利用 CPC 連結到條文，即可達成公司與條文間的連結。( 公司 -> 經濟部商業司營業項目登記 -> 主計處標準分類 -> CPC -> 條文 )
由於 CPC 分類、主計佈標準行業分類與公司行號營業代碼三者，有複雜的排容關係，並非單純的「一對一對一」，在資料庫設計上要特別小心；最後的使用者介面上，對於某些系統無法界定的分類，結果頁面也有「並非全部開放」的但書，以求精準嚴謹。

![](https://g0vhackmd.blob.core.windows.net/g0v-hackmd-images/upload_49d4a8279b50547bb61941534d4d88c2)
圖：結果頁面的「並非全部開放」但書。系統查得此公司有營業登記 F399040 無店面零售業，但無法得知這個公司的主要業務，是否符合但書裡提到的「行業標準分類」條件，故將之列出，讓使用者自行判斷是否受影響。

#### 2.是否有行銷網站的方式，好讓更多人知道這方便的工具？

服貿組目前沒有計劃，事實上，當初 beta 釋出時（3/30），也只有透過社交網路的個人帳號，以及貼到 g0v 後勤中心（Facebook 上的社團）而已，後來 g0v 粉絲專頁貼文（4/1），產生約兩千個分享，應是截至目前為止散佈網站的主要管道。

#### 5.官方怎麼會願意提供行業對照表(大眾普遍有政府黑箱的印象)？

這是反黑箱服貿陣營不斷透過各種管道（如公聽會、施壓立委等）要求，才有[初稿](http://tisa.g0v.tw/docs/mapping-draft.pdf)釋出。
政府依據政府資訊公開法本應提供，令人意外的是，行業對照表並非因為任何機密性的原因而延遲提供，而是自始缺乏，應該是行業對照 > 評估報告 \> 簽署，但實際上是反過來，參考[這裡](http://www.slideshare.net/nchild/g0v-remix)。
從這裡我們可以看到，政府當初簽署服貿、進行電訪評估時，對於服貿開放的各個項目其實是含混不明、無法清楚交代的。甚至到現在，行業對照表也仍然只是初稿。

洪偉解釋 g0v 運作方式：
Hackathon 黑客松活動裡，發起人提案 --> 現場找到有興趣的參與者，和服貿相關的專案會一起做事，所以是公開徵求參與。
自由參與，一個人先做一部分，另一個人接手前一個人的部份，核心貢獻者們慢慢把 code 完成
專案發起人會去看這個 project 的進程與方向。
Wikipedia 會讓所有貢獻者照單全收，而程式碼的協作則是在 github 上面，每個人都能複製一份出來改，但要將[複製版本（分支）](https://github.com/g0v/tisa-map/network)合併回主幹的話，專案管理人 Shelling 可以查看並決定是否要接受合併。
