**3.討論方式**
(1) 點子發想
- 長期 vision（主軸）
- 情境種類：用戶如何使用 SO、與 SO 互動來滿足主軸論述
- 實作種類 & 實作細節：具體能吸引、協助用戶使用 SO、與 SO 互動的方式
- 執行專案 & 人力：執行實作的方式

(2) 點子評估面向
- Example
![](https://s3-ap-northeast-1.amazonaws.com/g0v-hackmd-images/uploads/upload_2579c496dd91ba762ebb69f7a414bc8c.png)
- SO 版本
    - Benefit：
        - SO vision（專案面向的質化影響）
        - member engagement/Achievement（成員面向的質化影響）
        - funding pool（專案/成員面向的量化影響）
    - Cost: 
        - Dev effort
        - Design effort
        - Research effort
        - PJ effort
        - 教學 effort
        - 推廣 effort

(3) 點子決議/排優先級
- 以上面向權重相同，大家根據 1-5 給分
- 給完分後，先根據 benefit 挑出重要的實作
    - 可能是 benefit 分數高
    - 可能是 benefit 分數低但性質重要的任務，比如必要的 bug fix
- 針對初篩的實作，再搭配 cost 分數來評估最終要優先做哪些任務
