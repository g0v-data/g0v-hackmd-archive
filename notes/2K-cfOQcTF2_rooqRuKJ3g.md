# Ronny Brainstorming 憲法

本憲法為本系統之最高規範，所有議題均應遵守。

---

## 第一章　目錄結構

**第一條**　本系統之目錄結構如下：

```
法律/
  分類-{分類名稱}.md   # 各分類之共同規定
topics/
  {topic-id}/
    README.md          # 議題概覽、討論重點、里程碑
    insights.md        # 蒸餾後的關鍵洞察，為恢復 context 之主要依據
    施行細則.md         # 議題專屬規定（選擇性設立）
    conversations/
      YYYY-MM.md       # 每月對話紀錄（只增不減）
    references/        # 參考資料
README.md              # 所有議題索引（顯示於 GitHub 首頁）
憲法.md                # 本文件，全域規範
```

---

## 第二章　法律位階

**第二條**　本系統採三級規範架構，下位規範得補充或覆蓋上位規範：
1. **憲法**（本文件）：全域規範，適用於所有議題
2. **法律**（`法律/分類-{分類}.md`）：各分類之共同規定，適用於該分類下所有議題
3. **施行細則**（`topics/{topic-id}/施行細則.md`）：單一議題之專屬規定，選擇性設立

**第三條**　Claude 處理特定議題時，應依序讀取：憲法 → 對應分類之法律 → 議題施行細則（如存在）。

**第四條**　修憲程序：用戶說「我要修憲」時，修改本文件；修法程序：用戶說「我要修 {分類} 的法律」或「我要修 {topic-id} 的施行細則」時，修改對應檔案。Claude 亦得主動提出修憲或修法建議，經用戶確認後方可執行。

---

## 第三章　議題分類

**第五條**　每一議題應標註一個或多個分類。目前有效分類如下：

| 分類 | 說明 |
|---|---|
| 專案 | 有生命週期的行動項目，涵蓋發想、執行、結案全程 |
| 議題思辨 | 以討論、辯證、探索為主的思想性議題 |

**第六條**　分類為「專案」之議題，應標註狀態。有效狀態如下：**發想中**、**進行中**、**結案**。

**第七條**　Claude 發現現有分類不足以涵蓋某議題時，得提案修憲新增分類，經用戶確認後方可執行。

---

## 第四章　議題管理

**第七條**　議題識別碼（topic-id）應使用 kebab-case slug，力求描述性且簡潔（例：`ronny-brainstorming`、`product-strategy`）。

**第八條**　每一議題之 `README.md` 應包含：議題摘要、當前焦點、討論重點、待解問題、里程碑。

**第九條**　根目錄 `README.md` 每一議題記錄一列，包含：議題識別碼（含超連結）、標題、**分類**、**狀態**（專案類適用）、最後更新日期、一行摘要。

---

## 第五章　對話紀錄

**第十條**　每次對話結束前，應將完整對話 append 至 `topics/{topic-id}/conversations/YYYY-MM.md`，以 `## YYYY-MM-DD` 為段落標題。

**第十一條**　對話紀錄按月分檔，平時無須讀取；僅在追溯細節時方才查閱。

---

## 第六章　洞察摘要

**第十二條**　`insights.md` 記錄從對話中蒸餾出的關鍵洞察，每條洞察應標註日期。

**第十三條**　Claude 恢復舊議題 context 時，應讀取 `insights.md` 而非對話紀錄，以節省 token。

**第十四條**　每當對話中出現有意義的洞察，應即時更新 `insights.md`。

---

## 第七章　導航連結

**第十五條**　所有 `README.md` 應包含 GitHub 相對路徑超連結，使用者得從任一 README 移動至其他頁面，無須依賴檔案瀏覽器：
1. 根目錄 `README.md`：議題名稱連結至 `topics/{topic-id}/README.md`
2. 議題 `README.md`：頂部設「← Topics Index」連結至 `../../README.md`

---

## 第八章　版本控制

**第十六條**　每完成一個有意義的階段（新增議題、記錄對話、更新洞察、新增參考資料等），應自動執行：

```
git add -A
git commit -m "<簡述變更內容>"
git push
```

**第十七條**　Commit message 格式：
- 議題層級變更：`[{topic-id}] {變更內容}`
- 系統層級變更：`[system] {變更內容}`

---

## 第九章　Claude 行為準則

**第十八條**　Claude 應不定期重讀本憲法，以免遺忘規範。

**第十九條**　任何修憲行為，必須同步記錄於 `ronny-brainstorming` 議題之對話紀錄。

**第二十一條**　系統層級的重要決策（修憲、修法、新增分類等），除記錄對話外，亦須同步更新 `ronny-brainstorming` 議題之 `README.md` milestones 與 `insights.md`。

**第二十條**　Claude 發現更好的資料整理方式時，應主動提出，經用戶確認後方可執行。
