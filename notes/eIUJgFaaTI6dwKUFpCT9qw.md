## 二、每日匯報（Weekly Report）

* **時間**：每日 10:00am 前需完成。
* **範圍**：昨日10:00am到今日10:00am
* 如果有跨到假日則是上週五 10:00am 至本週一 10:00am 之間所有告警與事件。
* **重點**：

  1. 高風險告警列表與後續行動。
  2. MTTD / MTTR 指標趨勢。
  3. 改善建議（規則調整、白名單、IR 流程等）。

## 三、IP／網域調查流程

1. **初步判別（公司內部）**

   * 來源或目標屬 `172.16.0.0/16` = 總部網段。
   * 先查 **SRE 系統**：確認服務名稱與負責人。

2. **外部威脅情報三步驟**（皆在**私人電腦**上操作，避免企業來源洩漏）

   | 工具                                   | 用途                         | 快速指令 or 操作        |
   | ------------------------------------ | -------------------------- | ----------------- |
   | **VirusTotal**                       | Hash / IP / URL Reputation | Web UI 上傳或搜尋      |
   | **AbuseIPDB**                        | IP 信譽評分、舉報紀錄               | 貼入 IP → 檢視分數、Tags |
   | **LevelBlue** (IBM X-Force Exchange) | 威脅情資匯總                     | 搜尋 IP → 風險指數      |

   → 三方皆顯示高風險，即判定「惡意 IP」。

3. **網路診斷輔助**

   ```bash
   nslookup <domain>    # 查 A / AAAA / CNAME
   traceroute -a <IP>   # 帶 ASN，觀察流向
   ```

   *同樣僅用私用電腦執行。*

## 四、常見告警場景

| 現象                       | 可能原因              | 快速判斷                                  |
| ------------------------ | ----------------- | ------------------------------------- |
| 流量來自 **ngrok**           | 開發人員臨時暴露測試 API    | 先聯繫系統 owner，確認是否合法；若無，即封鎖 token / 端口。 |
| 目的地屬 **Amazon AWS**      | 雲端資源或惡意主機         | 依內網白名單判斷；不在清單則進一步 Whois、VT。           |
| **Forcepoint WSG** 報惡意網址 | Web Gateway 已阻擋外呼 | 拉出 WSG ➜ Internet 日誌交叉比對；確認是否誤判。      |

## 五、後續處置與外部協調

1. **端點深度掃描**

   * 使用 **Apex One Agent**：

     1. 在 **Search for Endpoint** 輸入電腦名稱 / IP。
     2. 選擇 *Task → Scan Now*（全機掃毒）。

2. **防火牆日誌**

   * 可查最外層（Edge）防火牆；若需內層日誌，提 Service Request。

3. **MDR 供應商（中華資安）協助**

   * 當三方威脅情報確認惡意、且內部流量難以溯源時：

     1. 整理事件摘要與 IOC。
     2. 透過 MDR Portal 或信件升級事件。

4. **提交 ITSR 表單**

   * 完成初步調查後，若確定為「Security Incident」：

     * 填寫 **ITSR (IT Security Report) Form**，附上時間線、來源 / 目標 IP、取證截圖與掃描結果。
     * 送簽 CSIRT / 資安長審核。

5. **Grok Pattern（可選）**

   * 對非標準 Syslog 建立 Grok Parser，使日志能正確進入 Elastic Index。

### 重點回顧

* **每日**：Dashboard 健康檢查 → 高中低告警盤點 → 事件調查與記錄。
* **每週一**：彙報上週五至今的事件、指標與改善項。
* **調查流程**：SRE 確認 ➜ 三方情報 ➜ 端點 / 防火牆核對 ➜ 必要時升級 MDR。
* **確認惡意** → 提交 **ITSR**，並更新 Playbook / 規則，防範重演。
