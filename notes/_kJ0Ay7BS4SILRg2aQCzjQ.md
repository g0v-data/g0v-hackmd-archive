vTaiwan 是一個致力於利用數位工具進行公共討論的社群，今年與臺灣網路資訊中心（TWNIC）合作，希望延續數位發展部三月發布的「人工智慧風險影響評估——兒少篇」中對於人工智慧對兒少風險的觀察，擇定「人工智慧生成之兒少不宜內容」作為深入討論的主題。

今年的 vTaiwan 一樣與臺灣網路資訊中心合作，舉辦審議大會活動討論科技治理相關議題！本次的審議大會議題從數發部今年三月發布的「人工智慧風險影響評估-兒少篇」出發，討論一個風險影響與頻率都很高的情境：「AI 生成／散佈兒少不宜內容」。我們的第一場活動會在 7/17 晚上舉辦，第二場會在 9/11 晚上進行。我們希望藉由這兩輪的活動，討論出人工智慧時代下，如何因應兒少內容的這一個風險！

當生成式 AI 讓內容創作與傳播變得更加容易，我們該如何界定平台、開發者、家長、教育工作者與政府各自應承擔的責任？面對可能對兒少造成傷害的內容，哪些預防、通報與治理措施最有效，同時又能兼顧隱私、創作自由與科技創新發展？你認為未來應該如何取得平衡？

在本次活動中，我們預計採取 vTaiwan 討論模式邀請多元利害關係人進行深入討論。方法為先以線上平台 polis 進行意見徵集後，邀請多元利害關係人參與後續的實體＋線上討論。在實體＋線上討論時，為了使實體與線上的參與者可以享有更好的討論品質，將會使用互動式投影片 mentimeter 與結合人工智慧的數位工具輔助進行討論！目前我們預計在台北舉辦兩場的討論，日期分別是 7/17 週五晚上 18:30-21:30，以及 9/11 週五晚上 18:30-21:30。

考慮到這是你和歐洲夥伴們開發的，我有對各建議附上 Google 機翻譯文供他們參考，若有表達不夠周全的地方還請幫我補充說明，感謝~ :pray:
#### Tim: 
根據實測結果，目前功能規劃有兩種身份：
(1) HOST 有開麥 & 打字 建立論點的權力，也可以分發 HOST 身份。
(2) 一般參與者僅能進行投票表態。
如果活動要順暢進行，收音品質跟辨識度滿關鍵的。我認為比較保險的做法，是活動當天準備一台手機跟一台筆電分別登入為 HOST。
活動當天將手機當作麥克風傳遞給現場參與者發表論點。如果手機成功辨識語音，筆電端 HOST 就直接 Approve 為論點；如果手機辨識語音不佳 (或是線上參與者)，那筆電的 HOST 就幫忙打字建立論點。
至於投票的結果呈現會跟主持人後續規劃流程有關，這部分 @Peter 也可以自行測看看提供想法~ :saluting_face:

(特別須留意的是 Vote Scale 分為 "Agree/Disagree" 跟 "Likert" 兩種模式，HOST 可以隨時切換，雖然方便但也會同步改變參與者的投票模式，HOST 需控管好避免影響投票結果。若議程有分階段投票的需求，或許另開新的 Session 避免干擾會是好方法。)
:::
