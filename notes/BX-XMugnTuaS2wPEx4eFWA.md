## 一般原則

### 藉此來推廣什麼？
- 認識開放資料集的種類
- 認識數位工具
    - 地理媒介：線上地圖、群眾標註地圖
    - 公民參與工具：Collage、Streetmix
    - 社群媒體的選擇
        - Line 官方帳號：一對多，每月 200 則訊息上限
        - Line 群組：
        - FB 社團：
        - g0v Slack：多頻道，可與其他既有社群頻道交流
        - Slack：多頻道，免費版的歷史檔案無法存取
        - Discord：
- 認識群眾外包的策畫方式
    - 有哪些角色、互動關係、哪一個環節想要開展協作貢獻、貢獻成果的授權 ...等
- 認識此地區可以馬上上手的「公民科學與群眾參與計畫」
    - 公民科學與群眾參與計畫清冊
        - https://g0v.hackmd.io/@chewei/crowdtribute/
        - 例如：挑出「苗栗縣-南庄生活圈」或「北投區-關渡平原」可運用銜接的計畫

### 列舉出「數位工具基礎建設」項目

#### 網站
- [ ] 建立一個入口網站 
    - [ ] 工具挑選：Google Site 或是 g0v.hackmd 書本模式
    - 可以考量中長期使用該工具，或已有？

#### 聯繫
- [ ] 建立一個訊息群組
    - [ ] 工具挑選：？ 或是 g0v slack
    - 可以考量中長期使用該群組，或已有？

#### 拍照
- [ ] 群眾標註拍照上傳地圖 - 建立關渡地區主題
    - 群眾標註工具網址 https://commutag.agawork.tw/
    - [ ] 建立表單欄位
    - [ ] 至 FB 社團提出建立需求 https://www.facebook.com/groups/2675853389340513

#### 此地區已有哪些「公民科學與群眾參與計畫」？
- [ ] 工作文件
    - https://g0v.hackmd.io/@chewei/crowdtribute/https%3A%2F%2Fg0v.hackmd.io%2F%40chewei%2Frk47XdX0i

#### 議題
- [ ] 整理 關渡地區議題爭點 issue
    - 從歷屆學生報告與提案中整理？
- [ ] 整理 關渡地區利害關係人角色 stakeholder
    - 從歷屆學生報告與提案中整理？
    - 主要是輔助於，發想群眾外包專案的時候，可以挑選 stakeholder 角色種類進行發想
